# 职责与角色

[← 返回规范主页](../规范.md)

**流程骨架按「职责」书写，职责由谁担任见「角色分配表」——切换协作模式只改分配表，骨架不动。**

## 职责定义（骨架，稳定层）

- **设计职责**：立案前查重（本仓 `views/all.md` + 已有需求）；定档提议（见[三档分级与分期模式](三档分级与分期模式.md)）；写蓝图/方案（含「给用户的摘要」、接口与数据设计、复用盘点、检查清单 YAML）；响应评审意见出 v+1；Gate 决策落地标注；文档状态流转与归档；验收后合入主线。
- **评审职责**：蓝图/方案/代码评审；事实独立复测（不采信他人自测）；独立建评审记录、轮次递增、不覆盖历史；只出结论与意见——不改代码、不改被评审文档、不立案（评审中发现的新需求写在评审页建议栏，由设计职责统一立案）。
- **实施职责**：按已通过的方案小步实施；每步本地 CI（lint / 测试 / 构建）全绿才 commit，message 标注步骤号；实施完成后**先跑预飞自检（见[流程、自检与 Gate](流程、自检与Gate.md)）再交评审**；只允许在方案页**追加**三类章节（实施记录 / 实施偏差记录 / 蓝图验证备忘）；不流转状态字段；发现不可行**立即停**，按变更协议登记后等待处置。
- **决策职责（用户）**：定档裁决、Gate 1（方案/蓝图放行）、Gate 2（验收）、偏差与冲突裁决、协作模式选择。

## 角色分配表（可变层）

| 职责 | 单 Agent 模式（默认） | 双角色模式 | 三角色模式 |
|------|------|-----------|-----------|
| 设计 | Agent-A | Agent-A | Agent-A |
| 实施 | Agent-A | Agent-A | Agent-C（如 codex）|
| 评审 | Agent-A 起草 / 用户把关 | Agent-B | Agent-B |
| 决策 | 用户 | 用户 | 用户 |

**当前生效模式：单 Agent 起草 + 用户 Gate**（个人自用、本地开发的轻量起步）：设计、实施由同一 session 承担；正式代码评审仍按独立评审职责输出**评审记录文档**，由设计 session 孵化的独立子代理担任（干净上下文，提示词强制：每个事实亲自复测、每章节至少一个可证伪质疑、附验证记录）。需要更强独立性时升双/三角色模式，只改本表，流程不动。

## 模型配置

| 职责 | 模型 | 理由 |
|------|------|------|
| A 设计 | 高配模型 | 方案质量所系，省不得 |
| B 评审 | **默认 sonnet** | 配合 checklist 脚本事实表后，评审以对照核查 + 定点判断为主；token 大头，降档收益最大 |
| 机械检查 | 脚本（preflight.py） | 不过 LLM 的手 |

**haiku 禁用于评审**——「可证伪质疑」需要的力度它撑不住，省的钱会从漏掉的问题里加倍吐出来。

任何分配必须满足的不变约束：评审职责与设计/实施职责**不得同一 agent 同一上下文**；设计与实施可合并；实施者权限按「职责定义」收窄；**交接只认文档 + git diff，对话内口头结论不算数**。
