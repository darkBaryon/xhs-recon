# 实施纪律与终扫

[← 返回规范主页](../规范.md)

## 变更协议（偏差即停）

实施职责只按已通过的方案做。一旦发现需要**改变目标、改变接口、改变步骤**，或方案某步实际不可行：

1. **立即停**——不要「先按自己的理解做下去再说」；
2. 在方案页「实施偏差记录」登记：哪一步、为什么、建议怎么办；
3. 等用户裁决（小偏差用户可口头放行并补登记；动到接口/目标的偏差须重新定档或出方案 v+1）。

漏做方案明示的条目同样是偏差。**「无偏差」是一个需要负责的断言**，不是默认值。

## 每期终扫（固定动作）

实施完成、预飞之前，跑一遍固定收尾，避免把脏东西交进评审：

- **调试残留清零**：临时 `print` / `console.log` / 断点 / 注释掉的旧代码 / `TODO(临时)` —— 写进 checklist 的「禁留标记」一并 grep 归零。
- **死代码与投机泛化清除**：本期为了功能加的、最后没用上的函数/参数/分支/配置项删掉；只为「以后可能用」预留的抽象删掉（要用时再加，成本更低）。
- **接口面对账**：实际新增的对外符号 ⊆ 方案 §5 声明；多出来的要么删、要么补进方案（走偏差登记）。
- **demo / 测试自跑**：交评审前自己先跑通 checklist 的必过命令。

## 硬规则

!!! rule "不可逾越"
    - 方案未通过前，**不写业务实现代码**（可写探针/草稿验证可行性，但不进主线、不算交付）。
    - 发现要改目标/接口/步骤，**先停下登记偏差**，不许先斩后奏。
    - 只读边界、敏感字段、外部访问约束等，**必须写进方案与 checklist**，预飞机械守护。
    - 本项目越界禁止项（代理池 / 验证码破解 / 自动点赞评论私信 / 保存评论用户敏感字段）写进「禁留标记」，预飞 0 命中才算合规。
    - 生成视图只通过 `build_views.py` 更新；带 `generated` 哨兵的文件手改一律视为无效。
