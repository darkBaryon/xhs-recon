## 阻塞项（不改会出错）

### 阻塞1：e2e 新增窗内断言用例须显式声明合法 ISO pin 值

**问题**：方案 §6 要求「新增用例：pin `collected_at` 下 jsonl 全部 `published_at` 落窗内 + `kept+out+missing==creator 总数`」。该用例只有在 `window_days > 0` 时才能测「出窗不入 feed」的验收口径。

亲核 `time_window.py:31-33`：
```python
now = _parse_iso(now_iso)       # '2026' → None
if now is None:
    raise ValueError("now_iso must be a valid ISO datetime")
```
`_parse_iso('2026')` 通过 `datetime.fromisoformat` 抛 ValueError 并返回 None（已验证）。因此，若实施者照期2 双配置测试 pin 手法（`monkeypatch _now_iso -> '2026'`）同时设 `window_days > 0`，测试在运行到 `filter_notes` 时直接崩溃（非断言失败，是 ValueError）。

方案未指明新用例应参考 `test_pipeline_window_filters_before_aggregate`（`pin='2026-07-02T00:00:00+00:00'`，window_days=30）而非期2 双配置（`pin='2026'`，window_days=0）。

**要求 v2 做的事**：在 §6 新增窗内断言用例描述处补充一句：「该用例的 `_now_iso` pin 须为合法 ISO（如 `'2026-07-03T00:00:00+00:00'`），不可复用期2 `'2026'` pin——`filter_notes(window_days>0, '2026')` 会 raise ValueError（`time_window.py:32-33`）；参照先例 `test_pipeline_window_filters_before_aggregate`。」
