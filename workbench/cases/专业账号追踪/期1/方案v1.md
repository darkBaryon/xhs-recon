## 给用户的摘要

期1 只解决「时间久远」：搜索改按发布时间排序、加时间窗过滤、选帖加时间衰减。

- Before：搜「留学辅导」→ 按热度排 → 2024 年的万赞爆款霸榜。
- After：同一条命令 → 按时间排 + 只留近 30 天（可配）→ 选帖新旧兼顾（互动 × 时间衰减）。

拍板清单：**无**（用量/默认值均已在蓝图 Gate 拍定；本期无新增取舍）。

## 1. 起点

`起点 = f727dc3`（xhs-recon 仓，src/ 干净；workbench 视图文件有未提交的文档改动，与代码无关）。MediaCrawler fork 独立 git 仓（lab 根 `../MediaCrawler`），其改动在该仓单独 commit，起点以实施当日 `git log -1` 为准记入实施记录。

## 2. 现状与约束盘点（写方案当时实测）

- 排序写死：`../MediaCrawler/config/xhs_config.py:24` `SORT_TYPE = "popularity_descending"`；消费点 `media_platform/xhs/core.py:155`；CLI 无 sort 参数（`cmd_arg/arg.py` 全表核过）。枚举 `media_platform/xhs/field.py:55-62`：`general / time_descending / popularity_descending`。
- adapter 命令构造：`src/adapters/mediacrawler_adapter.py:50-81` `_build_command`，构造参数在 `__init__:24-44`；注入点 `src/pipelines/run_research.py:28-44` `_build_adapter`。
- 聚合顺序：`src/core/aggregator.py:16-56` `aggregate(results)` 的 accounts **由 notes 重建**（nickname 另取）——**时间窗过滤放 aggregate 之前**，账号统计天然只算窗内笔记，无需改 aggregator。
- 时间字段：`Note.published_at` 已有（`src/models.py:25`），源自 `parsers.py:56` `epoch_ms_to_iso(row.get("time"))`；**缺失时为空串**（parser 既定降级）。
- 选帖：`src/core/note_selector.py:6-7` `_note_score = like + 2*collect + 3*comment`，纯互动、无时间。
- **fixture 陷阱**：`tests/fixtures/search_contents_sample.jsonl` 的 `time` 均为旧时间戳——若 demo 配置默认开窗，导出会被滤空。约束：`configs/sample.yaml` 与 `sample_mediacrawler.yaml` 的 `window_days` 默认 **0**（不过滤），保证既有 demo 导出**逐字节不变**；窗过滤行为由单测合成 Note 验证。
- **复用盘点**：时间解析复用 stdlib `datetime.fromisoformat`（published_at 已是 ISO 串）；过滤统计的输出暂用 `typer.echo`（`run_research.py:91-95` 已有 echo 模式）——不引日志（日志系统是另一个待计划需求，不在本期）；无现成时间过滤/衰减模块，`core/time_window.py` 为新建，无重复造轮。
- 约束：`core` 无 "xhs"、不碰 JSONL（铁律）；旧 5 导出文件契约不变；`ResearchAdapter.search` 端口签名不变（蓝图契约：排序是 adapter 构造参数）。

## 3. 目标 / 非目标

**目标**：

1. MediaCrawler fork CLI 新增**可选** `--sort_type`（缺省不覆盖 config 默认值），透传至 `config.SORT_TYPE`；
2. `MediaCrawlerAdapter` 支持 `sort_type` 构造参数并写入搜索命令；config 新增 `search.sort`；
3. 时间窗过滤（`search.window_days`，0=不过滤）：aggregate 前过滤，缺 `published_at` 归窗外，三类计数（kept / out_of_window / missing_time）echo 输出；
4. `note_selector` 时间衰减（`selection.half_life_days`，0=关闭=与现状逐字节一致）。

**非目标**：creator 拉取 / watchlist（期2）；专业度打分 / 新导出文件（期3）；不改旧 5 文件契约；不引日志框架；不改 `aggregate` / `account_ranker` / `parsers`。

## 4. 接口与数据设计

```python
#   --sort_type: str = ""   # 空串 = 不覆盖；非空时 config.SORT_TYPE = sort_type
#   轻校验：非空且不在 {general, time_descending, popularity_descending} 时报错退出

# --- xhs-recon ---
# src/adapters/mediacrawler_adapter.py
MediaCrawlerAdapter.__init__(..., sort_type: str = "")        # 新增构造参数，默认空=行为不变
# _build_command：sort_type 非空时追加 ["--sort_type", self.sort_type]

# src/core/time_window.py（新，平台无关，只认 models.Note）
class WindowFilterStats(BaseModel):
    kept: int; out_of_window: int; missing_time: int
def filter_notes(
    notes: list[Note], window_days: int, now_iso: str
) -> tuple[list[Note], WindowFilterStats]
# window_days <= 0：原样返回，stats=(len,0,0)
# published_at 为空串：missing_time += 1，不保留
# published_at < now - window_days：out_of_window += 1，不保留
# now_iso 由 pipeline 传 collected_at —— 纯函数、确定性、可测

# src/core/note_selector.py（签名扩展，默认值=现状）
def select_typical_notes(
    notes, top_per_account=2, half_life_days: int = 0, now_iso: str | None = None
) -> list[TypicalNote]
# half_life_days <= 0 或 now_iso 为 None：打分与现状完全一致（旧调用/旧测试零改动）
# 开启时：score = _note_score(n) * 0.5 ** (age_days / half_life_days)
#   age_days = (now - published_at).days；published_at 缺失按 age=365 计（惩罚不清零）
#   selection_reason = "top by interaction×recency"（开启时）

# src/pipelines/run_research.py（编排接线）
# aggregate 前：对每个 ok 的 FetchResult 过滤 r.notes 并累计 stats；echo 一行统计
# select_typical_notes 传入 half_life_days 与 collected_at

# config 契约（新增键，全部有默认值=旧行为）：
search:    { sort: "", window_days: 0 }        # sample.yaml（fixture）保持 0/空
selection: { half_life_days: 0 }
# sample_mediacrawler.yaml 示例值：sort: time_descending, window_days: 30, half_life_days: 14
```

依赖方向不变：`pipelines → core → models`；`time_window` 归 core（无平台知识）。

## 5. 实现步骤（小步，每步可独立 CI 全绿 commit）

1. **[MediaCrawler 仓]** `cmd_arg/arg.py` 加可选 `--sort_type` 透传 + 轻校验；手动 `--help` 核出参数（该仓无 CI，留 commit hash 入实施记录）；
2. `core/time_window.py` + `tests/test_time_window.py`（窗内/窗外/缺时间/0=不过滤 四组）；
3. `note_selector` 衰减扩展 + 测试（half_life=0 与旧断言完全一致；开启时新帖胜旧爆款；缺时间惩罚）；
4. `MediaCrawlerAdapter` 加 `sort_type` + 命令构造测试（非空含 `--sort_type`，空不含）；
5. pipeline 接线（过滤 + stats echo + 配置读取）+ 两份 sample 配置更新 + e2e 测试（fixture 配置 0 值 → 导出与既有逐字节一致）。

## 6. 测试策略

- 单测（行为断言）：
  - `time_window`：窗内保留 / 窗外剔除并计数 / 空 `published_at` 计 missing_time / `window_days=0` 原样通过；
  - `note_selector`：`half_life_days=0` 时输出与现有测试期望完全一致（回归）；开启时「7 天前 100 赞」胜「400 天前 1 万赞」（half_life=14 可复算）；缺时间按 age=365 惩罚；
  - `mediacrawler_adapter`：`sort_type="time_descending"` 时命令含该参、空串不含（注入 `_run_crawler` 不真跑）；
  - e2e：`sample.yaml`（全 0 默认）跑通且 `data/exports/*` 与起点逐字节一致。
- 手动验收（不可自动化）：真实跑 `sample_mediacrawler.yaml`（sort=time_descending, window=30）一次，抽查导出 notes 的 `published_at` 新近性 + echo 统计行合理。
- 盲区：fork `--sort_type` 在 MediaCrawler 侧无自动化测试（该仓不进 CI），靠 `--help` 核对 + 真实跑覆盖。

## 7. 验收标准 + 检查清单 YAML

机械核验以同目录 [checklist.yaml](checklist.yaml) 为唯一事实源：lint/format/pytest/fixture-demo 全绿；`time_window.filter_notes`、adapter `--sort_type`、`half_life_days` 符号落地；core 无 "xhs"、src 无 `print(`。人话解释：默认配置全 0 → 一切旧行为与旧导出不变；新能力全部 opt-in。

## 8. 风险与禁止事项

- **fixture demo 与窗过滤的默认值耦合**：sample.yaml 必须保持 `window_days: 0`，否则 demo 导出被滤空、checklist 必存产物挂——已在 §2 约束钉死；
- **双仓提交**：fork 改动在 MediaCrawler 仓、其余在 xhs-recon 仓——实施记录须分别记 hash，代码评审两边都查；
- **时区**：`published_at` 与 `collected_at` 均 UTC ISO（parsers/pipeline 既定），`time_window` 统一 `fromisoformat` 比较，不引入本地时区；
- 禁止：动 `aggregate`/`account_ranker`/`parsers`/旧导出列；`--sort_type` 之外的 fork 改动；开并发。

## 9. 回滚口径

未合主线前弃功能分支即可；MediaCrawler 仓的 `--sort_type` commit 独立、单 revert 干净（可选参数缺省不改变任何现有行为，留着也无害）。已 commit 的去留由用户裁决。
