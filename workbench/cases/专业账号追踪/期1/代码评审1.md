### 质疑1【已关闭】：window_days=0 时 pipeline 不输出 echo 统计行，是否违反方案 §3「三类计数 echo 输出」？

**可证伪点**：若方案要求任何配置下都 echo，则 `_apply_time_window` 的 `if window_days <= 0: return results`（先于 echo）漏输出。

**验证**：亲读 §3 目标3 原文——「时间窗过滤（`search.window_days`，0=不过滤）：aggregate 前过滤，……三类计数 echo 输出」，echo 从属于「过滤」语境；§4 伪码同。0=不过滤时无统计可言；且全 0 demo 输出保持与起点一致（不添噪声行）反而更贴「默认行为不变」。e2e 对 window=30 断言了 stdout 恰为一行统计。**结论**：实现取舍与方案语境一致，非漏做，关闭。

### 质疑2【已关闭】：`_report_note_ids` 正则若在报告上匹配不到任何 URL，投影断言会以「空集 == 空集」虚真——正则真的匹配产物吗？

**验证**：对真实 `data/exports/report_input.md` 亲跑该正则 → 5 个 note_id（`664176cd…`/`666c1412…` 等）。非虚断言，关闭。

### 质疑3【已关闭】：`filter_notes` 对非法 `now_iso` 抛 ValueError 是方案外行为，会不会在真实管线炸？

**验证**：`collected_at` 唯一来源 `run_research.py` `_now_iso()` = `datetime.now(tz=utc).isoformat()`，恒为合法 aware ISO；`parsers.epoch_ms_to_iso` 产出同为 aware UTC（或空串走 missing_time 路径），无 naive/aware 混减的 TypeError 可能。该分支仅防手工误用，纯函数防御，不计偏差，关闭。

### 质疑4【已关闭】：commit 2c5ddc9 中与 sort_type 无关的测试字符串重排是否夹带？

**验证**：提取 f727dc3 版 `tests/test_mediacrawler_adapter.py` 单文件跑 `uv run ruff format --check` → `Would reformat`（exit 1）；`ruff format` 对旧文件的唯一改写 diff 恰等于该 hunk。即当前锁定的 ruff 版本要求此格式，改动是维持 `format` 门禁全绿的必要动作，非顺手美化。关闭。（侧记：这意味着起点 f727dc3 在当前环境下 format 检查本会挂，疑为 ruff 版本漂移所致，与本期实现无关。）
