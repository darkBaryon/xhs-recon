## 给用户的摘要

期1 只解决「时间久远」：搜索改按发布时间排序、加时间窗过滤、选帖加时间衰减。

- Before：搜「留学辅导」→ 按热度排 → 2024 年的万赞爆款霸榜。
- After：同一条命令 → 按时间排 + 只留近 30 天（可配）→ 选帖新旧兼顾（互动 × 时间衰减）。

拍板清单：**无**（用量/默认值均已在蓝图 Gate 拍定；本期无新增取舍）。

## 2. 现状与约束盘点（写方案当时实测，v2 经评审 #1 逐条核对）

- 排序写死：`../MediaCrawler/config/xhs_config.py:24` `SORT_TYPE = "popularity_descending"`；消费点 `media_platform/xhs/core.py:155`；CLI 无 sort 参数（`cmd_arg/arg.py` 全表核过）。枚举 `media_platform/xhs/field.py:55-62`：`general / time_descending / popularity_descending`。空串传入 `core.py:155` 会回落 `GENERAL`——所以 adapter 侧「空=不传参」而非「空=传空串」。
- adapter 命令构造：`src/adapters/mediacrawler_adapter.py:50-81` `_build_command`，构造参数在 `__init__:24-44`；注入点 `src/pipelines/run_research.py:28-44` `_build_adapter`。
- 聚合顺序：`src/core/aggregator.py:16-56` `aggregate(results)` 的 accounts **由 notes 重建**（nickname 另取）——**时间窗过滤放 aggregate 之前**，账号统计天然只算窗内笔记，无需改 aggregator。
- 时间字段：`Note.published_at` 已有（`src/models.py:25`），源自 `parsers.py:56` `epoch_ms_to_iso(row.get("time"))`；**缺失时为空串**（parser 既定降级）。
- 选帖：`src/core/note_selector.py:6-7` `_note_score = like + 2*collect + 3*comment`，纯互动、无时间。
- **fixture 时间分布（实测更正）**：`tests/fixtures/search_contents_sample.jsonl` 5 条 `time` 混布——含 2026-06-20（本日窗内）与 2026-04-01 等窗外项。**开窗（如 30 天）不会滤空，但会改变导出内容**，破坏「默认行为不变」的回归基线；空数据下 `exporter.py` 亦写表头文件，必存产物不受影响。约束不变：两份 sample 配置 `window_days` 默认 **0**（不过滤），旧行为回归由 e2e 双配置对比守住（§6）；窗过滤行为由单测合成 Note 验证。
- **复用盘点**：时间解析复用 stdlib `datetime.fromisoformat`（published_at 已是 ISO 串）；过滤统计输出暂用 `typer.echo`（`run_research.py:91-95` 已有 echo 模式）——不引日志（日志系统是另一个待计划需求，不在本期）；无现成时间过滤/衰减模块，`core/time_window.py` 为新建，无重复造轮；e2e 回归复用期3 的 pin 时间戳手法（`tests/test_pipeline_e2e.py:78-107`）。
- 约束：`core` 无 "xhs"、不碰 JSONL（铁律）；旧 5 导出文件契约不变；`ResearchAdapter.search` 端口签名不变（蓝图契约：排序是 adapter 构造参数）。

## 3. 目标 / 非目标

**目标**：

1. MediaCrawler fork CLI 新增**可选** `--sort_type`（缺省不覆盖 config 默认值），透传至 `config.SORT_TYPE`；
2. `MediaCrawlerAdapter` 支持 `sort_type` 构造参数并写入搜索命令；config 新增 `search.sort`；
3. 时间窗过滤（`search.window_days`，0=不过滤）：aggregate 前过滤，缺 `published_at` 归窗外，三类计数（kept / out_of_window / missing_time）echo 输出；
4. `note_selector` 时间衰减（`selection.half_life_days`，0=关闭=与现状逐字节一致）。

**非目标**：creator 拉取 / watchlist（期2）；专业度打分 / 新导出文件（期3）；不改旧 5 文件契约；不引日志框架；不改 `aggregate` / `account_ranker` / `parsers`（机械兜底见 checklist）。

## 6. 测试策略

- 单测（行为断言）：
  - `time_window`：窗内保留 / 窗外剔除并计数 / **age == window 边界保留（含等号）** / 空 `published_at` 计 missing_time / `window_days=0` 原样通过；
  - `note_selector`：`half_life_days=0` 时输出与现有测试期望完全一致（回归）；开启时「7 天前 100 赞」胜「400 天前 1 万赞」（half_life=14 可复算）；缺时间按 age=365 惩罚；**未来 published_at 得分不高于当天发布（clamp 断言）**；
  - `mediacrawler_adapter`：`sort_type="time_descending"` 时命令含该参、空串不含（注入 `_run_crawler` 不真跑）；
  - e2e 回归（复用期3 pin 手法 `tests/test_pipeline_e2e.py:78-107`）：pin `collected_at` 后，**全 0 配置的导出与起点行为一致**（对固定 fixture 断言既有期望值）；**开窗配置（如 window=3650/半衰开）导出为全 0 配置的确定性子集/重排**——两组断言均机械可验。
- 手动验收（不可自动化）：真实跑 `sample_mediacrawler.yaml`（sort=time_descending, window=30）一次，抽查导出 notes 的 `published_at` 新近性 + echo 统计行合理。
- 盲区：fork `--sort_type` 在 MediaCrawler 侧无自动化测试（该仓不进 CI），靠 `--help` 核对 + 真实跑覆盖。

## 7. 验收标准 + 检查清单 YAML

机械核验以同目录 [checklist.yaml](checklist.yaml) 为唯一事实源：lint/format/pytest/fixture-demo 全绿；`time_window.filter_notes`、adapter `--sort_type`、`half_life_days` 符号落地；core 无 "xhs"、src 无 `print(`；**`aggregate`/`account_ranker`/`parsers` 对起点零 diff（git diff --quiet 兜底）**。人话解释：默认配置全 0 → 一切旧行为不变；新能力全部 opt-in。

## 8. 风险与禁止事项

- **fixture 与窗过滤的默认值耦合**：sample.yaml 必须保持 `window_days: 0`，否则开窗改变导出内容、e2e 回归基线破（v2 更正：非「滤空挂产物」）——已在 §2 钉死；
- **双仓提交**：fork 改动在 MediaCrawler 仓、其余在 xhs-recon 仓——实施记录须分别记 hash，代码评审两边都查；
- **时区**：`published_at` 与 `collected_at` 均 UTC ISO（parsers/pipeline 既定），`time_window` 统一 `fromisoformat` 比较，不引入本地时区；
- 禁止：动 `aggregate`/`account_ranker`/`parsers`/旧导出列；`--sort_type` 之外的 fork 改动；开并发。

## 9. 回滚口径

未合主线前弃功能分支即可；MediaCrawler 仓的 `--sort_type` commit 独立、单 revert 干净（可选参数缺省不改变任何现有行为，留着也无害）。已 commit 的去留由用户裁决。
