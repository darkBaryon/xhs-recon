## 给用户的摘要

期2 交付「公开创作者内容同步与 watchlist 配置」：把 config 里的手动账号（user_id 或主页 URL）与搜索自动发现的 top-N 账号合成 watchlist，用 MediaCrawler 既有 creator 模式逐账号拉取最新 N 篇公开发帖，导出 `watchlist.csv` + `creator_notes.csv` 两个新文件；旧 5 文件契约与旧行为零变化（无 `watchlist` 配置段 = 完全跳过）。

- Before：账号只是热帖副产品——知道「陈皮糖」值得盯，但每次只能靠关键词搜索碰运气。
- After：config 写一行 manual 账号（或让 auto 发现补足）→ 一条命令产出每个 watchlist 账号的最新 10 篇发帖清单。

拍板清单：

- **非法 manual 项处理**：提议 fail-fast（无法归一化的配置项直接报错退出，改完再跑）——利：配置手误不会静默丢账号；弊：一条写错整跑中断。备选是跳过并计数警告。

## 2. 现状与约束盘点（写方案当时实测）

- **spike 事实（2026-07-02 真实拉取成功）**：`--type creator --creator_id <纯 24 位 hex user_id>` 可用，无需 URL/xsec_token 降级路径；落盘 `<save_path>/xhs/jsonl/creator_contents_*.jsonl`，21 字段与 search 完全一致 → `parsers` 全复用；返回按主页最新发布顺序，数量由 `--crawler_max_notes_count` 控制（creator 流无 20 下限钳制，蓝图 §3 知情项）；`liked/collected/comment/share` 四个互动字段为空串、`source_keyword` 为空串（见 [spike-creator.md](spike-creator.md)）。
- 端口先例：`ResearchAdapter.fetch_comments` 非抽象、基类默认 `raise NotImplementedError`（`src/core/ports.py:17-21`）——`fetch_creator_notes` 照此模式，旧 adapter 不被强制实现。
- `FetchResult`（`src/models.py:57-76`）已含 `operation/notes/raw_path/error/command` 字段，creator 结果直接复用，无需新模型；`operation` 现有取值 `"search"`/`"fetch_comments"`（`src/adapters/mediacrawler_adapter.py:191,209`）。
- 命令构造先例：`MediaCrawlerAdapter._build_command`（`mediacrawler_adapter.py:55-88`），构造参数含 `sort_type`（期1加，`:86-87`）；search JSONL glob `xhs/jsonl/search_contents_*.jsonl`（`:136`）——creator 改 glob `creator_contents_*.jsonl` 即可。
- **复用盘点**：JSONL 解析复用 `parsers.parse_jsonl_lines`（`src/adapters/parsers.py:84`，`_row_keyword` 的 fallback 参数天然支持 creator 无来源词场景 `:40-41`）；互动数空串走 parser 既定降级（转 0）；CSV 导出复用 `exporter._write_csv`（`src/core/exporter.py:18-23`）；auto 发现直接消费 `rank_accounts` 现有输出 `list[AccountRank]`（`src/core/account_ranker.py:16`），不新增打分逻辑；URL 解析 src/ 下无现成工具，`normalize_creator_ref` 为新建、无重复造轮。

- 约束（铁律不变）：`core` 无 "xhs"、不碰原始 JSONL——「主页 URL→user_id」归一化归 `adapters.parsers`，`core.watchlist` 只吃归一化 id（蓝图 §2）；旧 5 文件导出契约逐字节口径不变（只新增）；`aggregate`/`account_ranker` 本期零 diff（机械兜底见 checklist；注意 `parsers` 本期要动，不进 no-touch 清单）；只读拉取、低频、单并发、人工授权运行，不做定时任务。

## 3. 目标 / 非目标

**目标**：

1. 端口 `ResearchAdapter.fetch_creator_notes`（基类默认 NotImplementedError）+ MediaCrawler / fixture 双实现；
2. `parsers.normalize_creator_ref`：manual 项归一化（纯 user_id 或主页 URL → user_id）；
3. `core/watchlist.py`：manual + auto top-N 合成、去重、限量（新模型 `WatchAccount`）；
4. 导出 `watchlist.csv` + `creator_notes.csv`（旧 5 文件不动）；
5. pipeline 接线：config 新增 `watchlist`/`creator` 段，缺段 = 全跳过（opt-in）；
6. 顺手项：`log_result` DEBUG 补记 `FetchResult.command`（期1 已知项 1）。

**非目标**：专业度打分 / `account_profile.csv` / `topic_feed`（期3）；**不追加单篇详情调用补互动数**（用户已定口径：互动信号复用 search 命中的 notes，creator 列表互动列 0=未采集）；creator notes 不进 `aggregate`/`rank_accounts`（空互动会污染统计，仅供 `creator_notes.csv` 导出与期3 消费）；不做置顶笔记识别（蓝图既定，留余量兜底）；不改 MediaCrawler fork；不加并发、代理、定时任务。

## 6. 测试策略

- 单测（行为断言优先）：
  - `normalize_creator_ref`：五组见 §5-2，非法输入断言 `ValueError` 消息含原始 ref；
  - `build_watchlist`：manual 优先去重、截断、K/N 不硬凑、空 ranked 仅 manual 也成立；
  - `FixtureAdapter.fetch_creator_notes`：按 account_ids 过滤、limit 截断、`creator_path=None` 抛 NotImplementedError、`operation=="creator_notes"`；
  - `MediaCrawlerAdapter`：命令含 `--type creator --creator_id`、每账号独立 save_path、limit 透传 `--crawler_max_notes_count`、部分失败时 notes 保留且 error 列出失败 id；
  - `exporter`：不传新参时目录中**不出现**两新文件（旧行为回归）；传入时列头与行数正确；
  - pipeline：无 `watchlist` 段全跳过（不调 fetch_creator_notes，mock 断言）；manual 非法 ref 报错退出；adapter 无 creator 能力时降级空表头。
- e2e（`tests/test_pipeline_e2e.py` 扩展，pin `collected_at` 手法照旧 `:78-107`）：sample.yaml 开 watchlist 段走 fixture 全管线 → 两新文件存在且内容确定；**旧 5 文件与不开 watchlist 时逐字节一致**（creator notes 不进 aggregate 的机械证明）。
- 手动验收（Gate 2，不可自动化）：真实跑一次——manual 指定 1 账号（可用 spike 账号）+ auto 发现补足，抽查 `creator_notes.csv` 每账号条数 ≤ notes_per_account 且发布时间新近、`watchlist.csv` source 标注正确。
- 盲区：MediaCrawler creator 子进程真跑不进 CI（同期1 口径，靠手动验收覆盖）；置顶笔记挤占名额无法在 fixture 层验证（蓝图既定风险，留余量兜底）。

## 8. 风险与禁止事项

- **置顶笔记挤占最新名额**：creator 流按主页顺序截断，置顶旧帖会占 N 篇名额——`notes_per_account: 10` 默认含余量，期3 topic_feed 窗过滤兜底；本期不做置顶识别（蓝图既定）。
- **manual URL 形态**：只支持 `/user/profile/<id>` 路径与纯 id 两种形态，其余 fail-fast 报错（拍板项）——形态不够用是配置问题，报错信息带原始 ref 便于自查。
- **多账号串行时长**：10 账号 × 每账号一次子进程会话，串行单并发；低频、人工授权触发，不做定时任务（蓝图既定节奏）。
- **部分账号拉取失败**：口径已钉死（§4）——成功保留、失败入 error 汇总，pipeline 警告不中断，导出以成功部分为准。
- 禁止：动 `aggregate`/`account_ranker`/旧导出 5 文件列；追加单篇详情调用；MediaCrawler fork 任何改动；并发放大、代理、定时任务、`--sort_type` 之外的行为外溢。
