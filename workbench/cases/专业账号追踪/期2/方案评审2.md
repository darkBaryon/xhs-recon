## 差量新问题排查

### 新问题检查1：creator_fixture_path 与 _build_adapter 现有结构的衔接

v2 §4 钉死 `creator_fixture_path` 为顶层键，`_build_adapter` 注入 `FixtureAdapter(creator_path=...)`。

现有 `_build_adapter`（`run_research.py:34-52`）只有两条路径，均末尾 `return FixtureAdapter(config["fixture_path"], comments_path=comments_path)`——**当前 FixtureAdapter 构造不含 `creator_path` 参数**。这是方案设计的增量（期2要实施），并非 v2 引入的新矛盾；方案已在 §4 明确「`FixtureAdapter.__init__(..., creator_path: str | None = None)` 新增可选构造参数」。结构衔接方案清晰，无新问题。

**可证伪质疑 Q-V3（新，建议级）**：`_build_adapter` 当前在 `provider == "mediacrawler"` 分支下不创建 FixtureAdapter（创建 MediaCrawlerAdapter），但 v2 §4 说「`_build_adapter` 传 `FixtureAdapter(creator_path=...)`」——这里「传」的是 fixture provider 路径。然而如果 provider 是 `mediacrawler`，`creator_fixture_path` 是否也要透传给 `MediaCrawlerAdapter`？v2 未对 mediacrawler provider 场景下 `creator_fixture_path` 的行为有任何说明（`MediaCrawlerAdapter.fetch_creator_notes` 真实拉取，不需要夹具路径，这是合理的）。但 `_build_adapter` 只创建 adapter 对象，后续 creator 逻辑在 pipeline 层——**v2 §4 pipeline 步骤4 调 `adapter.fetch_creator_notes`，不区分 fixture/mediacrawler provider**。建议 §4 或 §5-4 步骤对「fixture provider 时传 creator_path、mediacrawler provider 时无需」显式说清，避免实施时在 `_build_adapter` 里对 mediacrawler 分支也意外注入夹具参数。（不构成阻塞，设计意图可从上下文推断，但多走一步明确更好。）

### 新问题检查2：§4 端口规格注释与蓝图契约一致性

v2 §4 端口规格注释（第78-79行）：「部分账号失败时 notes 保留成功部分、error 汇总失败 id——即 ok=False 且 notes 非空是 creator 场景的**正常态**，调用方不得因 ok=False 丢弃 notes。」

这段注释是 creator 场景**专属语义扩展**，附在 `ResearchAdapter.fetch_creator_notes` 的接口注释上。核查与蓝图契约关系：蓝图 §3 接口契约定义了 `FetchResult.ok = error is None`，未对「ok=False 时 notes 可能非空」做特殊限制——v2 是在蓝图已留出的语义空间内定案，不与蓝图冲突。✅

### 新问题检查3：checklist 必含符号覆盖

v2 新增的接口面（`WatchAccount`、`fetch_creator_notes`、`build_watchlist`、`normalize_creator_ref`、`creator_notes.csv`、`fetch_creator_notes`（adapter）、`creator_path`（fixture）、`watchlist`（pipeline））全部在 checklist 必含符号8条里，无遗漏。v2 变更未新增 checklist 覆盖死角。✅

### 新问题检查4：v2 变更章节与正文的一致性

v2 §「v1→v2 变更」第3条写「顶层 `creator_fixture_path`（`configs/sample.yaml:2`）」——这里的 `sample.yaml:2` 是说「与现有 `fixture_path` 所在的 `configs/sample.yaml:2` 平级」，不是说 `creator_fixture_path` 本身会在第2行。措辞略有歧义，但 §4 正文第137行注释已明确「顶层键、与既有 `fixture_path`（`configs/sample.yaml:2`）平级」，上下文可消歧。
