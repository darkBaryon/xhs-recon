## 给用户的摘要

期2 交付「公开创作者内容同步与 watchlist 配置」：把 config 里的手动账号（user_id 或主页 URL）与搜索自动发现的 top-N 账号合成 watchlist，用 MediaCrawler 既有 creator 模式逐账号拉取最新 N 篇公开发帖，导出 `watchlist.csv` + `creator_notes.csv` 两个新文件；旧 5 文件契约与旧行为零变化（无 `watchlist` 配置段 = 完全跳过）。

- Before：账号只是热帖副产品——知道「陈皮糖」值得盯，但每次只能靠关键词搜索碰运气。
- After：config 写一行 manual 账号（或让 auto 发现补足）→ 一条命令产出每个 watchlist 账号的最新 10 篇发帖清单。

拍板清单：

- **非法 manual 项处理**：提议 fail-fast（无法归一化的配置项直接报错退出，改完再跑）——利：配置手误不会静默丢账号；弊：一条写错整跑中断。备选是跳过并计数警告。

## 8. 风险与禁止事项

- **置顶笔记挤占最新名额**：creator 流按主页顺序截断，置顶旧帖会占 N 篇名额——`notes_per_account: 10` 默认含余量，期3 topic_feed 窗过滤兜底；本期不做置顶识别（蓝图既定）。
- **manual URL 形态**：只支持 `/user/profile/<id>` 路径与纯 id 两种形态，其余 fail-fast 报错（拍板项）——报错由 pipeline 捕获打印、消息含原始 ref，便于自查后重跑。
- **多账号串行时长**：10 账号 × 每账号一次子进程会话，串行单并发；低频、人工授权触发，不做定时任务（蓝图既定节奏）。
- **部分账号拉取失败**：口径已钉死（§4 端口规格 + pipeline 步骤 4）——成功保留、失败入 error 汇总，调用方不论 ok 值消费 notes、warning 不中断，导出以成功部分为准；adapter 侧与 pipeline 侧各有单测锁定（§6）。
- 禁止：动 `aggregate`/`account_ranker`/旧导出 5 文件列；追加单篇详情调用；MediaCrawler fork 任何改动；并发放大、代理、定时任务。
