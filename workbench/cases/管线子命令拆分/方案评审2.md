## 逐条对账

### 1. 阻塞1（sync 空表头覆盖 latest 自断链路）是否关闭

**推演 search → sync → comments 三命令序列（补全式写回设计）**：

1. `cli search`：`_search_stage → select_typical_notes → export_all`（watchlist 侧参数全 `None`）→ 产出 `accounts/notes/account_rank/typical_notes/report_input` 五件（真实数据）→ 新建 `run_dir`（复用 `run_research._compact_run_id`/`_update_latest_link` 的命名与软链逻辑，模块属性可达，见验证记录）→ `latest` 首次指向该目录。
2. `cli sync`：`run_dir = resolve_latest_run_dir(...)` → resolve 到 search 的目录（真实数据，未被覆盖）→ `ranks = load_ranks_csv(run_dir/account_rank.csv)`（读到 search 写的真实榜单，`auto` 生效）→ `_sync_stage → export_watch_side(run_dir, ...)`——**只在同一目录内新增/覆盖四件**（`watchlist/creator_notes/account_profile/topic_feed`），不建新目录、不碰 `latest` 软链、不动已有五件。
3. `cli comments`：`run_dir = resolve_latest_run_dir(...)` → 仍 resolve 到同一目录（`latest` 全程未被 sync 动过）→ `typical = load_typical_csv(run_dir/typical_notes.csv)` 读到 search 写的**真实**典型笔记（非空表头）→ `_comments_stage → export_comments(run_dir, ...)` 补 `comments.csv` + 重写 `report_input.md`。

三步终态：同一目录内 11 件产物齐全，`latest` 全程指向同一实际目录。**v1 阻塞1 的死循环（sync 覆盖 latest → comments 读到空 typical → 报错）在 v2 设计下不成立**——因为 v2 的 sync 不再新建目录/不再抢占 latest，序列自洽。✅ 关闭。

**边角① 用户直接 `cli sync`（无任何历史跑）**：`resolve_latest_run_dir` 遇「缺失/悬空/非目录」→ `ValueError`，§4 明确「报错退出：先跑 search」，口径清楚、不静默降级。✅

**边角② `cli research` 之后再 `cli sync` → 覆盖自己四件**：`research` 内部调用未改的 `run_research()`（原样六阶段+`export_all`全量写），产出完整 11 件（含 watch 侧四件、含 comments.csv 如启用）。随后 `cli sync` 的 `export_watch_side` 只重写四件 watch 侧文件，不碰 search 侧五件、不碰 `comments.csv`/`report_input.md`——与拍板清单「同命令重复跑覆盖自己那部分产物属预期」口径完全一致（`research` 写的 watch 侧四件视为「上一次 sync 的等价物」，被新 `sync` 覆盖符合「最新同步状态」语义）。✅ 一致，无矛盾。

**边角③ exporter.py 移出 no-touch 的三处一致性**：

## 结论

方案评审 #1 的三处阻塞在 v2 中均被针对性关闭，且关闭方式经得起亲核推演：

1. **阻塞1**：v1「sync 新建目录+空表头覆盖 latest」的自毁链路被 v2「补全式写回」（sync/comments 都写回 latest 指向的既有目录、不新建目录不动软链）替换，三命令序列推演终态自洽；两个用户边角场景（直接 sync 无历史跑 / research 后再 sync）口径均与拍板项一致；exporter 子集出口的提取式重构在 §1/§7/checklist 三处口径一致，且有单测+三层 e2e 字节比对的机械验证路径。
2. **阻塞2**：`_split_pipe` 空值规则钉死在 §4/§5/§6 三处，`"".split("|")==['']` 的坑经本轮亲测确认成立，规则本身正确闭合该坑。
3. **阻塞3**：cli 子命令统一走模块属性访问 `run_research._now_iso()`，pin 目标与既有测试手法（`tests/test_pipeline_e2e.py:115`，本轮亲核属实）同构，Python 名字绑定语义上必然生效。

建议1/2 均已在对应正文章节（非仅摘要）落地。v1→v2 的文本 diff 逐段可追溯到声明的 5 点变更，未见夹带新功能/新接口；两处观察（stale 榜单风险披露、search 建目录措辞显式化）均属阻塞1修复的自然延伸，非范围外内容。

发现两处非阻塞级缝隙（`__init__.py` 漏检、`cli search` 建目录复用哪个私有函数未点名），均登记为建议栏第3/4条，不影响本轮通过，交实施期落实即可。

verdict **通过**，可进入实施。
