## 给用户的摘要

把单体管线拆成四个独立子命令：`search`（每周广角搜索）、`sync`（每周 watchlist 同步）、`comments`（按需深读评论）、`research`（三段组合=现行为）。命令之间靠 `data/exports/latest/` 的既有 CSV 文件衔接，不引数据库不加配置段；旧入口与旧导出契约逐字节不变。

- Before：想只同步 watchlist 新帖也得全跑 40 分钟；评论想单独补采做不到。
- After：`cli sync` 8-10 分钟出 topic_feed；评论只在要做深度分析时 `cli comments` 补一跑。

拍板清单：

- **comments 写回原跑目录**：提议 `comments` 命令把 `comments.csv` 与重写的 `report_input.md` 写回 latest 指向的那次运行目录（补全语义）——利：报告和它的数据在同一目录；弊：破「按运行归档不覆盖」的字面惯例（仅重写 report_input.md 一个文件）。备选是新建目录但报告与源数据分家。

## 1. 起点

`起点 = e9640d1`（xhs-recon 仓 main；src/ 干净）。本期**只动 `src/pipelines/`**：`core`/`adapters`/`models` 全零 diff（checklist 机械兜底）；MediaCrawler fork 零改动。B1/B2 修复、评论上限、creator 单会话优化均属下一案（非目标）。

## 2. 现状与约束盘点（写方案当时实测，行号基于 e9640d1）

- 单体编排：`run_research()`（`src/pipelines/run_research.py:139-297`）内联六阶段——搜索循环 `:162-177`、窗过滤+聚合+打分 `:179-184`、watchlist/creator/profile/feed `:186-243`、选典型 `:245-253`、评论 `:255-275`、导出+软链 `:277-296`。阶段间通过局部变量传递，无函数边界——拆分即把这些段落抽成阶段函数，`run_research` 变组合调用（行为零变化）。
- CLI 现状：typer 单命令 `main`（`:314-321`），入口 `python -m src.pipelines.run_research`；README/run.sh/两期 checklist 都引用此入口，**必须保持可用**。
- 跨跑衔接的文件契约现成：`account_rank.csv` 列 `account_id,nickname,...,account_score,note_ids`（`src/core/exporter.py:227-250`）、`typical_notes.csv` 列与 `TypicalNote` 字段一一对应（`:251-258`）——读回反序列化无信息缺失（`AccountRank` 期3 分项字段有默认值，`src/models.py:46-48`）。
- `latest` 软链维护：`_update_latest_link`（`run_research.py:300-311`），`Path.exists()` 跟随软链（期2 评审亲核）——`sync`/`comments` 读 `latest/` 即读最近一跑。
- 日志：`configure_logging` 以 `run_id=collected_at` 配置（`:148-153`），可重复调用（`_remove_own_handlers`，`logging_setup.py:71-75`）——每个子命令独立跑各配各的，无冲突。
- **复用盘点**：阶段逻辑全部已在（core 纯函数 + adapter 端口），本期零新算法；CSV 写出契约在 exporter，读回函数无现成实现——新建 `pipelines/artifacts.py`（组装层职责：文件契约 ↔ 模型，`core` 不感知文件布局）；typer 子命令模式为标准库能力，无重复造轮。
- 约束：旧 5+6 件导出契约逐字节口径不变；`research` 与旧入口等价（e2e 字节对比机械可验）；不加配置段；`core` 无 "xhs" 铁律不变。

## 3. 目标 / 非目标

**目标**：

1. `run_research.py` 阶段函数抽取（`_search_stage`/`_sync_stage`/`_comments_stage`/`_export_stage`），`run_research()` 变组合，行为零变化；
2. 新增 `src/pipelines/cli.py`：typer 子命令 `search`/`sync`/`comments`/`research`，入口 `python -m src.pipelines.cli <子命令> --config ...`；
3. 新增 `src/pipelines/artifacts.py`：`load_ranks_csv`/`load_typical_csv`（latest 文件契约 → 模型）；
4. `sync` 独立跑：auto 名额读 `latest/account_rank.csv`，缺文件 → 警告 + 纯 manual（降级机械可验）；
5. `comments` 独立跑：读 `latest/typical_notes.csv` + `latest/account_rank.csv`，采评论后**写回该跑目录**（新增 `comments.csv`、重写 `report_input.md`，其余文件不碰——拍板项口径）；
6. README 快速开始补子命令用法。

**非目标**：不动 `core`/`adapters`/`models`（B1 search 落盘隔离、B2 读回包 try、B5/B6 隐患、评论批量上限、creator 单会话均属下一案）；不加并发/定时任务/新配置段；旧入口 `run_research` 模块与其 CLI 不删不改语义；`search` 子命令不采评论（评论归 `comments`）。

## 5. 实现步骤（小步，每步可独立 CI 全绿 commit）

1. 阶段函数抽取（纯重构）：`_search_stage`/`_sync_stage`/`_comments_stage` + `SyncArtifacts`；既有全部测试零改动通过 = 行为零变化的机械证明；
2. `artifacts.py` + 单测（rank/typical CSV 写出→读回往返一致；缺列/缺文件 ValueError）；
3. `cli.py` + `research`/`search` 子命令 + 测试（**research 与旧入口同 fixture 配置导出逐字节一致**；search 不产 watchlist 侧文件）；
4. `sync` 子命令 + 测试（有榜单 → auto 生效；缺榜单 → 警告 + 纯 manual；导出含四件 watchlist 侧文件）；
5. `comments` 子命令 + 测试（对已有跑补齐 `comments.csv` + `report_input.md` 含评论；其余文件字节不变；latest 缺失报错）+ README 更新。

## 6. 测试策略

- 单测：`artifacts` 往返与错误路径；`SyncArtifacts` 默认；
- e2e（pin `_now_iso` 手法照旧）：
  - `cli research` vs 旧入口 `run_research()`：同 fixture 配置双跑，全部导出文件**逐字节一致**（等价性的机械证明）；
  - `cli search`：产旧 5 件、不产 watchlist/creator/profile/feed 四件；
  - `cli sync`：预置一个含 `account_rank.csv` 的 latest 跑 → auto 名额生效；删掉榜单 → 警告降级纯 manual；
  - `cli comments`：预置含 `typical_notes.csv` 的跑 → 补齐后 `comments.csv` 存在、`report_input.md` 含评论行、**其余文件字节不变**；
- 手动验收（Gate 2）：真实配置跑一次 `cli sync`（授权触发），确认 8-10 分钟出 topic_feed 且质量与全流程一致。
- 盲区：真实 MediaCrawler 路径不进 CI（既定口径）。

## 8. 风险与禁止事项

- **comments 写回原跑目录**（拍板项）：仅重写 `report_input.md` + 新增 `comments.csv`，其余文件禁碰——测试以「其余文件字节不变」锁定；
- **文件契约反序列化依赖列稳定**：`account_rank.csv`/`typical_notes.csv` 列是既有导出契约（两期评审守护），`artifacts` 缺列即 ValueError fail-fast，不静默补默认；
- **sync 的旧 5 件空表头**：sync 单独跑无搜索数据，旧 5 件以空表头落盘（与期2 降级口径同形态）——`latest` 会指向这个「不完整」跑，comments 此时无 typical 可读会明确报错，属预期链路（README 写明推荐顺序 search → sync → comments）；
- 禁止：动 core/adapters/models；改旧导出列；加配置段；并发/定时；在本案夹带 B1/B2 修复（下一案的事，评审按范围哲学把关）。
