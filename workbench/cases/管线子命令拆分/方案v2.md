## 给用户的摘要

把单体管线拆成四个独立子命令：`search`（每周广角搜索，建当次运行目录）、`sync`（每周 watchlist 同步，**补全写回**最近一跑）、`comments`（按需深读评论，同样补全写回）、`research`（三段组合=现行为）。命令间靠 `data/exports/latest/` 目录逐步补全衔接——search 建目录，sync 补四件，comments 补评论与报告；不引数据库不加配置段，旧入口与旧导出契约逐字节不变。

- Before：想只同步 watchlist 新帖也得全跑 40 分钟；评论想单独补采做不到。
- After：`cli sync` 8-10 分钟出 topic_feed；评论只在要深度分析时 `cli comments` 补一跑；三个命令的产物始终在同一目录里。

拍板清单：

- **sync/comments 补全式写回**：两个命令都写回 `latest` 指向的那次运行目录（sync 新增/覆盖 watchlist 侧四件；comments 新增 `comments.csv`、重写 `report_input.md`）——利：一次研究的所有产物在同一目录、latest 语义不变、推荐链路自洽；弊：目录不再是「单命令一次成型」，同命令重复跑会覆盖自己那部分产物（仅限自己那部分，他人产物禁碰、测试锁定）。备选是每命令各建目录，但 latest 会指向残缺跑、链路自断（评审 #1 阻塞1 已证伪该设计）。

## v1→v2 变更（供差量评审）

回应「方案评审 #1」（verdict 需修改，3 阻塞 + 2 建议）：

1. **关闭阻塞1（sync 空表头覆盖 latest 自断链路）**：v1 的「sync 新建目录 + 旧 5 件空表头」被评审推演证伪（sync 会把 latest 指到无 typical 的跑，comments 必报错）。v2 改为**补全式写回**：sync 不新建目录、不动 latest 软链，把 watchlist 侧四件写进 latest 指向目录；latest 不存在/无效 → 报错退出（提示先跑 search 或 research）。为此 exporter 抽两个公开子集出口（见 §4，`export_all` 委托之，导出契约零变化、等价性 e2e 字节守护），no-touch 白名单相应把 `exporter.py` 移出、其余 core 文件保持零 diff。
2. **关闭阻塞2（`note_ids` 空列表反序列化形变）**：§4 钉死 `_split_pipe` 规则——空串 → `[]`，非空 → `split("|")`（评审实测 `"".split("|") == ['']` 的坑）；§6 单测点名「空 note_ids 往返仍为 `[]`」。
3. **关闭阻塞3（`_now_iso` patch 口径）**：§4 钉死 cli 各命令一律以**模块属性**访问 `run_research._now_iso()`（`import` 模块而非 `from ... import` 符号），新 e2e 的 pin 目标与既有手法同为 `"src.pipelines.run_research._now_iso"`（`tests/test_pipeline_e2e.py:115` 先例）。
4. **吸收建议1**：§5 开头补「每步 CI 全绿指 pytest/ruff 套件；checklist 全量（含 demo-cli）在步骤 3 后才可跑，交评审前以 checklist 全量终检为准」。
5. **吸收建议2**：`SyncArtifacts` 钉死落 `run_research.py`（与阶段函数同文件，组装层私有）。

## 3. 目标 / 非目标

**目标**：

1. `run_research.py` 阶段函数抽取（`_search_stage`/`_sync_stage`/`_comments_stage` + `SyncArtifacts` 同文件私有），`run_research()` 变组合，行为零变化；
2. `exporter` 提取两个公开子集出口：`export_watch_side`（watchlist/creator_notes/account_profile/topic_feed 四件）、`export_comments`（comments.csv + report_input.md），`export_all` 委托之，列与内容零变化；
3. 新增 `src/pipelines/cli.py`：typer 子命令 `search`/`sync`/`comments`/`research`；
4. 新增 `src/pipelines/artifacts.py`：`load_ranks_csv`/`load_typical_csv`（含 `_split_pipe` 空值规则）+ `resolve_latest_run_dir`；
5. `sync`/`comments` 补全式写回 latest 目录（拍板项口径）；`sync` 的 auto 名额读该目录 `account_rank.csv`（缺/空 → 警告 + 纯 manual）；
6. README 快速开始补子命令用法与推荐顺序。

**非目标**：不动 `core`（除 exporter 提取式重构）/`adapters`/`models`；exporter 不改任何列、不加新文件种类；B1/B2、B5/B6、评论上限、creator 单会话属下一案；不加并发/定时/新配置段；旧入口不删不改语义；`search` 子命令不采评论。

## 8. 风险与禁止事项

- **补全写回的覆盖边界**（拍板项）：sync 只写四件、comments 只写 `comments.csv`+`report_input.md`，各自禁碰他人产物——e2e「其余文件字节不变」锁定；同命令重复跑覆盖自己产物属预期（最新同步状态）。
- **exporter 重构回归面**：提取必须原样搬代码（列/顺序/格式禁改），等价性 e2e + 既有 exporter 测试双保险；评审按 diff 逐行核「纯移动」。
- **文件契约反序列化**：缺列即 ValueError fail-fast，不静默补默认；`_split_pipe` 空值规则单测锁定。
- **stale 榜单**：sync 读上一次 search 的榜单做 auto，榜单可能是上周的——属预期（auto 语义=「最近一次全景扫描的头部」），README 写明。
- 禁止：动 core 其余文件/adapters/models；改导出列；加配置段；并发/定时；夹带 B1/B2 修复（下一案）。
