# 多平台模块化架构升级 方案评审 #2

**评审对象**：`开发蓝图.md` v1 → `开发蓝图v2.md` 差量

## 结论

**verdict：通过。** v1 评审的 4 个阻塞项均已在 v2 以可实施、可验证的契约关闭；差量未引入新阻塞。

## 四个阻塞项关闭核查

| # | v1 阻塞 | v2 差量证据 | 结论 |
|---|---|---|---|
| 1 | `Platform StrEnum` 使真实新平台必改 domain | §3.1 改为正则校验的开放 `PlatformId`；`XHS` 常量明确归 `platforms/xhs` | **已关闭** |
| 2 | §3.1 双向投影与 §5.1 只允许旧→新矛盾 | §2.3 区分“兼容桥可双向转换数据流”与“新 domain/application 不 import legacy”；§5.1 重申业务规则只存在新 domain | **已关闭** |
| 3 | 期 3 切回旧 repository 看不到切换后数据 | §3.5 增加停写水位、v2→legacy 幂等反向同步、全套对账与失败禁止回退；期 3 验收要求切换后新数据回旧库无缺口 | **已关闭** |
| 4 | 期 2 切新用例，但 output/repository 实现到期 3 才出现 | §3.4 明列 `LegacyXhsRepositoryBridge`、`LegacyXhsFileOutput`、`LegacyEntrypointShim`，并将三者纳入期 2 交付和旧行为验收 | **已关闭** |

## 逐章差量核查

### 给用户的摘要

摘要已将“平台名是开放值对象”和“停写+双向迁移脚本回滚”纳入用户可见拍板项，与后文一致。

- **Q0**：摘要的“新增平台不修改领域核心”是否仍会被固定枚举证伪？§3.1 已无平台枚举，任意合法字符串可构造 `PlatformId`，**质疑已关闭**。

### §1 整体目标 / 非目标

新目标明确第二平台 fixture 使用任意字符串身份，非目标明确不删旧表和不复制业务规则，未扩大到真实抖音实现。

- **Q1**：期 1 的 `test_social` fixture 能否不修改 domain 参与同一搜索契约？§1.1(8) 与期 1 验收同时明确了修改白名单与共用契约，**可由建筑测试和 contract test 直接证伪，无方案缺口**。

### §2 模块边界与依赖方向

v2 新增 `infrastructure/compatibility` 并显式画出兼容桥对旧模块和新契约的依赖，解决了 v1 将数据流与 import 方向混用的问题。

- **Q2**：兼容桥的反向数据投影是否会迫使新 application import 旧 `src.models`？§2.3 禁线与 §5.1 架构测试口径均明确禁止，转换所有权在外层兼容桥，**质疑已关闭**。

### §3 接口契约草案

`PlatformId`、细粒度能力、三个用例的 `RunReceipt` 返回契约、期 2 兼容实现及数据库正反迁移均已闭合。源码抽查确认 `src/core/store.py` 定义 `Store`，`src/adapters/mysql_store.py` 定义 `MySQLStore(Store)` 并提供幂等 upsert，兼容 repository 桥有真实旧端口可承接。

- **Q3**：§3.4 声称的旧 repository 承载者是否实际存在，且能接受桥转换后的旧 DTO？亲核 `Store` 方法与 `MySQLStore` 的 `upsert_accounts/notes/comments/profiles`，**存在且契约可承接**。

### §4 分期表

期 2 已将三个兼容实现与用例切换同批交付；期 3 已将正反迁移幂等性、切换后新数据无缺口列为验收条件。

- **Q4**：期 2 切换入口时是否还依赖期 3 才实现的 v2 repository/output？期 2 已完整列出 legacy repository/output/entrypoint 桥，数据 schema 不变，**质疑已关闭**。

### §5 风险与禁止事项

v2 将停写水位、正反迁移和摘要对账纳入风险控制，并新增“无停写水位与对账禁止切换/回滚”的硬约束。

- **Q5**：反向同步或对账失败时是否仍可以切回旧 repository？§3.5 明示规定失败就保持 v2，§5.2 又将无对账回滚列为禁止事项，**质疑已关闭**。

### 蓝图验证备忘

备忘已追踪本轮修订的开放身份、兼容桥业务规则禁线与正反迁移无缺口。

- **Q6**：实施中若改变回滚协议，是否会绕过方案重审？备忘明确要求改变跨期契约或回滚协议时出 v3，**有明确 Gate**。

## 新增阻塞检查

未发现 v2 修订引入新阻塞。`PlatformPlugin.config_model` 仍位于外层，不违反 domain 的 Pydantic 禁线；反向迁移只承诺本需求范围内的 XHS 数据，与“不实现真实第二平台”一致。
