# 多平台模块化架构升级 期1 方案 v1

## 给用户的摘要

期1只建立**完全离线的新系统闭环**：在 `src/recon/` 新建领域模型、用例、能力端口、平台注册、XHS fixture、新配置/CLI、内存存储和新版文件输出；旧系统一行不改，真实采集/MySQL/Web 都不碰。

- Before：当前 fixture、算法和编排仍围绕裸 `note_id/account_id` 与宽泛 adapter 生长。
- After：新入口可离线跑通 search/track/research；实体自带 platform；第二测试平台无需修改核心即可接入；新包机械保证不 import 旧业务包。
- 本期用表征测试复用旧算法语义，不做兼容壳，也不删除旧代码。

拍板清单：**无**。目录、开放身份、新输出不兼容和分期已在蓝图 v5 Gate 决定。

## 1. 起点

`起点 = 07eda553812bfea08c24d881fd6430e51ce09992`。

工作树已有用户修改 `configs/留学辅导/run.yaml` 和未跟踪 `data/`，以及本案工作台文档/生成视图；实施不得修改或清理用户配置与现有 `data/`。新运行产物统一写 `data/recon/`，新配置写 `configs/recon/`，与旧系统隔离。

代码盘点：`src/` 现有约 4053 行；本期只新增 `src/recon/`、`tests/recon/`、`configs/recon/`，除必要的包/工具配置外不改旧业务实现。

## 2. 现状与约束盘点

### 2.1 可复用资产

- 领域输入形状：`tests/fixtures/search_contents_sample.jsonl`、`creator_contents_sample.jsonl`、`creator_creators_sample.jsonl`、`comments.jsonl` 已覆盖搜索、创作者、档案和评论。
- 解析行为：`src/adapters/parsers.py:1-203` 已锁定中文互动数、时间、标签、主页引用和评论字段转换；本期新 XHS translator 用同 fixture 做表征对照，不在新包 import 它。
- 聚合：`src/core/aggregator.py:16-56` 按内容 ID 去重、合并来源词、派生创作者统计。
- 排名/画像：`src/core/account_ranker.py:27-96` 已有排名、垂直度和活跃度公式。
- 时间与选典型：`src/core/time_window.py:25-55`、`src/core/note_selector.py:29-66` 已有稳定专项测试。
- watchlist：`src/core/watchlist.py:13-68` 已锁定 self/manual/auto 顺序、去重与上限。
- 流程样本：`tests/test_pipeline_e2e.py`、`test_fixture_adapter.py` 等可作为新旧同样本投影依据。

复用方式是把行为迁到新模型并对照测试；`src/recon` 运行时禁止 import 上述旧模块。

### 2.2 现状问题

- `src/models.py` 使用裸 `note_id/account_id`，跨平台同 ID 会冲突。
- `src/core/ports.py:13-26` 是宽泛 `ResearchAdapter`，实际增量能力未完整入端口。
- `src/pipelines/run_research.py` 同时负责编排、能力探测、存储和输出，且直接导入平台 parser。
- `src/pipelines/config.py` 把通用配置与 MediaCrawler 配置混在同一模型。

### 2.3 本期约束

- 纯离线：禁止网络、CDP、MediaCrawler 子进程、MySQL 和真实文件媒体下载。
- 新旧隔离：旧 `src.models/core/adapters/pipelines`、旧配置、旧输出、旧测试不修改；完整旧 pytest 必须继续全绿。
- 平台隔离：`domain/application` 禁止出现 `xhs/douyin/mediacrawler` 或平台条件分支。
- 依赖隔离：整个 `src/recon` 禁止 import 旧业务包。
- 不做兼容：不解析旧 YAML，不生成旧 CSV，不增加旧入口 shim。
- 数据安全：所有新 demo 产物只写 `data/recon/`。

## 3. 目标 / 非目标

### 3.1 目标

1. 建立标准库 dataclass 领域模型与开放平台身份。
2. 建立 search/creator-feed/detail/comments 四个细粒度采集端口、repository/output/clock 端口和平台 registry。
3. 迁移关键词扩展、聚合、排名/画像、时间窗、watchlist、典型内容选择规则。
4. 实现 `SearchResearch`、`TrackCreators`、`RunResearch` 三个同步用例。
5. 实现 XHS fixture plugin，读取现有四类 fixture 并翻译为新领域模型。
6. 实现测试专用 `test_social` plugin，证明任意合法平台名可注册并运行相同 search contract。
7. 实现内存 repository、新版 manifest+平台分区文件输出、Pydantic 新配置与 Typer 新 CLI。
8. 新入口离线跑通 search/track/research，同时旧全量测试继续通过。

### 3.2 非目标

- 不实现 MediaCrawler/live XHS、进度条、图片归档和 backfill-media（期2）。
- 不实现 MySQL、增量轮转、change-log、历史迁移和 recovery（期3/4）。
- 不实现 bundle、FeedQuery、静态 Web（期5）。
- 不修改 `run.sh` 或日常 `configs/留学辅导/`（期6）。
- 不删除或重命名任何旧代码/测试（期7）。
- 不迁移旧输出格式，不保证新旧文件逐字节一致；只对业务投影做表征对照。

## 4. 接口与数据设计

### 4.1 领域模型

`src/recon/domain/identity.py`：

```python
@dataclass(frozen=True, slots=True)
class PlatformId:
    value: str                    # __post_init__: ^[a-z][a-z0-9_-]{1,31}$

@dataclass(frozen=True, slots=True)
class EntityId:
    platform: PlatformId
    external_id: str              # strip 后非空
```

`src/recon/domain/content.py`：

```python
@dataclass(frozen=True, slots=True)
class Engagement:
    likes: int = 0; collects: int = 0; comments: int = 0; shares: int = 0

@dataclass(frozen=True, slots=True)
class MediaRef:
    kind: Literal["image", "video"]
    remote_url: str = ""
    local_path: str = ""

@dataclass(frozen=True, slots=True)
class Creator:
    id: EntityId; nickname: str; source_queries: tuple[str, ...] = ()

@dataclass(frozen=True, slots=True)
class Content:
    id: EntityId; creator_id: EntityId
    title: str; body: str; tags: tuple[str, ...]; url: str
    engagement: Engagement
    published_at: datetime | None; collected_at: datetime
    source_queries: tuple[str, ...] = (); media: tuple[MediaRef, ...] = ()

@dataclass(frozen=True, slots=True)
class Comment:
    id: EntityId; content_id: EntityId; parent_id: EntityId | None
    body: str; engagement: Engagement; created_at: datetime | None; collected_at: datetime

@dataclass(frozen=True, slots=True)
class CreatorProfile:
    creator_id: EntityId; nickname: str = ""; bio: str = ""; followers: int = 0
    verification: str = "unknown"; collected_at: datetime | None = None
```

所有跨实体引用构造时校验 platform 相同；数值不得为负。平台专属 `red_id/verify_type` 不进通用模型，fixture translator 可映射为通用 verification，原始字段留 raw artifact。

`src/recon/domain/research.py` 定义 `CreatorRank/WatchTarget/TypicalContent/ResearchRun/RunReceipt`；ID 均为 `EntityId`，序列字段用 tuple，运行结果包含 `run_id/platforms/failures`。

### 4.2 采集与插件契约

`src/recon/application/ports/collection.py`：

```python
@dataclass(frozen=True)
class CollectionFailure:
    code: str; subject: str; message: str; retryable: bool = False

@dataclass(frozen=True)
class CollectionResult:
    platform: PlatformId; operation: str
    contents: tuple[Content, ...] = (); creators: tuple[Creator, ...] = ()
    comments: tuple[Comment, ...] = (); profiles: tuple[CreatorProfile, ...] = ()
    failures: tuple[CollectionFailure, ...] = (); raw_artifacts: tuple[str, ...] = ()

class SearchProvider(Protocol):
    def search(self, request: SearchRequest) -> CollectionResult: ...
class CreatorFeedProvider(Protocol):
    def list_creator_content(self, request: CreatorFeedRequest) -> CollectionResult: ...
class ContentDetailProvider(Protocol):
    def fetch_details(self, request: DetailRequest) -> CollectionResult: ...
class CommentProvider(Protocol):
    def fetch_comments(self, request: CommentRequest) -> CollectionResult: ...
```

`PlatformServices` 以可选字段显式装能力。用例构造/执行前调用统一 `require(capability)`，缺能力抛 `CapabilityUnavailable`；禁止 `hasattr()`。

`PlatformPlugin` 暴露 `name/config_model/build(config,deps)`；`PlatformRegistry.register/build` 拒绝重名、未知名和 name≠services.platform。生产 registry 只显式注册 XHS fixture plugin；`test_social` 在测试 composition root 注册，证明核心无需新增枚举。

### 4.3 Repository、Output、Clock

```python
class ResearchRepository(Protocol):
    def save(self, run: ResearchRun) -> None: ...
    def known_content_ids(self, platform: PlatformId) -> frozenset[EntityId]: ...
    def contents(self, platforms: frozenset[PlatformId] | None = None) -> tuple[Content, ...]: ...

class ResearchOutput(Protocol):
    def write(self, run: ResearchRun) -> Mapping[str, Path]: ...

class Clock(Protocol):
    def now(self) -> datetime: ...
```

期1 `InMemoryResearchRepository` 按 `EntityId` 幂等覆盖；不实现跨进程增量状态。`SystemClock` 只在 entrypoint 注入；测试用 `FixedClock`。

### 4.4 用例

- `SearchResearch.execute(SearchCommand)`：扩词 → 对每平台/关键词调用 search → 时间窗 → 聚合 → 排名 → 选典型 → repository.save → output.write → `RunReceipt`。
- `TrackCreators.execute(TrackCommand)`：按 `WatchTarget.platform` 分组 → require creator-feed → 拉 fixture creator 内容/档案/评论 → 画像/选典型 → save/output。
- `RunResearch.execute(ResearchCommand)`：组合公开的 search/track 服务返回的内存 `ResearchRun`，不得调用私有函数；同 run 只写 repository/output 一次。
- 失败语义：平台配置/能力缺失 fail fast；单次 fixture 文件读取失败进入 `CollectionFailure`，其余平台可继续，receipt 明示 failures。

### 4.5 业务规则迁移

- 聚合 key 从裸 ID 改为 `EntityId`；来源词保序去重。
- 排名公式保持 `10*内容数 + 5*关键词命中 + 0.01*平均互动`；画像保持 `10*垂直度 + 1*窗内内容数`。
- 典型内容保持 `likes + 2*collects + 3*comments` 与可选半衰期。
- 时间窗缺发布时间时不保留并计 missing；window_days<=0 原样。
- watchlist 保持 self→manual→auto、去重、self 不被 max_total 截断。
- 旧新对照只比较稳定业务投影（ID、排序、分数、计数），不比较类型名、文件名或序列化布局。

### 4.6 XHS fixture 与配置

`src/recon/platforms/xhs/fixtures.py` 直接解析现有 JSONL；`translators.py` 负责互动数、时间、标签、评论/档案映射。fixture provider 显式实现四项能力，不访问网络。

新配置 `configs/recon/sample.yaml`：

```yaml
schema_version: 1
platforms:
  xhs:
    collector: fixture
    fixture:
      search: tests/fixtures/search_contents_sample.jsonl
      creator: tests/fixtures/creator_contents_sample.jsonl
      profiles: tests/fixtures/creator_creators_sample.jsonl
      comments: tests/fixtures/comments.jsonl
jobs:
  search: {platforms: [xhs], keywords: [留学辅导], pages: 1, limit: 20, window_days: 0}
  track: {targets: [{platform: xhs, external_id: "601d0481000000000101cc46"}], limit: 10}
ranking: {weights: {note_count: 10, keyword_hit: 5, interaction: 0.01}}
selection: {top_per_creator: 2, half_life_days: 0}
storage: {driver: memory}
output: {root: data/recon/exports}
```

`src/recon/entrypoints/config.py` 用 Pydantic 校验通用段，再由 registry 的 `config_model` 校验平台段；未知字段 forbid，schema_version 非 1 fail fast。

### 4.7 CLI 与新输出

正式命令：

```text
python -m src.recon.entrypoints.cli search   --config configs/recon/sample.yaml
python -m src.recon.entrypoints.cli track    --config configs/recon/sample.yaml
python -m src.recon.entrypoints.cli research --config configs/recon/sample.yaml
```

期1文件输出写 `data/recon/exports/<run_id>/manifest.json` 与 `<platform>/creators.csv|contents.csv|comments.csv|creator_ranks.csv|typical_contents.csv`，并维护 `latest` 软链。manifest 至少含 schema_version/run_id/created_at/platforms/counts/failures/files。字段是新契约，不保旧列。

## 6. 测试策略

### 6.1 自动测试

- 模型：非法 platform/external_id、跨平台引用、负互动、timezone-naive datetime 均拒绝。
- 端口/registry：重复注册、未知平台、缺能力、plugin name 不一致显式失败。
- policies：复用现有测试向量；新旧投影断言聚合 ID/来源词、排名/画像分数、窗口计数、watchlist 顺序、典型内容排序一致。
- translators：现有 XHS 四类 fixture 映射；损坏 JSON 行形成 failure，不泄漏平台字段到 domain。
- contract：XHS fixture 与 `test_social` 共用 SearchProvider contract；同裸 ID 的两个平台聚合后为两个实体。
- use cases：search/track/research 正常、部分失败、能力缺失；research repository/output 各调用一次。
- output：manifest 与分区文件一致，counts/files 可对账，两个平台目录隔离，latest 指向本次。
- CLI：CliRunner 三命令退出 0；未知 schema/platform/config 错误退出非 0。
- 架构：新 core 零平台词；`src/recon` 零旧业务 import；application/domain 零外层依赖。
- 回归：现有全量 pytest 保持全绿。

### 6.2 验收用例

输入 `configs/recon/sample.yaml`：

- `search` 产生 XHS 搜索结果、排名和典型内容；
- `track` 产生指定创作者内容、档案和评论；
- `research` 合成两侧结果，只持久化/输出一次；
- `latest/manifest.json` 可读取，列出的文件全部存在且 counts 与 CSV 行数一致；
- `tests/recon` 中注册 `test_social` 后，同一个 external ID 与 XHS 不冲突。

### 6.3 盲区

- 不验证真实 MediaCrawler、MySQL、图片、进度、Web/bundle。
- 只证明旧算法在选定测试投影上等价，不保证旧代码所有未测试分支被迁移。
- 内存 repository 不证明跨运行增量语义；第3期另立 contract。

## 7. 验收标准 + 检查清单 YAML

同目录 `checklist.yaml` 是唯一机械事实源：新旧 lint/format/pytest、三命令 demo、架构禁线和工作台校验全部通过；新包/配置/manifest/平台文件存在；关键接口符号落地。

人话口径：不看目录“像不像新架构”，而看第二平台能否零核心改动接入、同 ID 是否不冲突、新入口是否完整离线跑通、旧系统是否完全未受影响。

## 8. 风险与禁止事项

- **范围膨胀**：期1模块多但只有离线风险域；任何 live/MySQL/Web/run.sh 工作立即登记偏差并停止。
- **伪复用**：禁止新包 import 旧函数；表征测试可在 tests 边界同时 import 新旧。
- **领域泛化**：只迁移现有用例消费字段；平台私有字段不得塞进万能 metadata。
- **可变状态**：domain dataclass frozen；集合 tuple；业务规则返回新值，不原地改实体。
- **输出污染**：禁止写旧 `data/exports`、`data/media`，只写 `data/recon`。
- **采集合规**：禁止网络、CDP、MySQL、代理池、验证码破解、自动互动、并发放大。
- **调试残留**：禁止 `print`、断点、临时 TODO、注释掉旧代码。

## 9. 回滚口径

本期只新增旁路目录和 gitignore 运行产物，旧入口/配置/代码不改。未合主线时弃本期 commits 即回到起点；`data/recon/` 可直接删除，不影响旧数据。若发现必须修改旧业务实现才能跑通，视为方案不可行，停止并回蓝图/方案重审。
