# 多平台模块化架构升级 期3 方案 v1

## 给用户的摘要

watchlist 成为与 account/search 平级的独立接口：只读 YAML 手工账号，不读取搜索排名；
按数据库刷新时间选择到期账号，先读主页列表，再只抓未知帖详情。每账号独立保存，
单账号失败不影响已完成账号，也不误标成功。

拍板项：无。功能独立原则沿用用户 Gate；真实采集继续暂不执行。

## 1. 起点

Git 起点 `07eda553812bfea08c24d881fd6430e51ce09992`，承接期1/2工作树；用户配置不修改。

## 2. 现状与约束盘点

旧 track 会把 search 排名的 auto 账号与 manual 合并，属于组合流程耦合；但已有成熟的
主页列表、详情、评论、图片、轮转时间和进度事件可复用。独立 watchlist 只消费
`watchlist.manual`，`auto_top_n` 留给未来 research 组合器。

## 3. 目标 / 非目标

目标：独立 WatchlistCollector/SyncWatchlist/WatchlistRepository/WatchlistOutput；支持
refresh_days、batch_size、列表 diff、新帖详情、可选评论、每账号事务、日志和终端状态。

非目标：不读取 search 输出，不跑真实采集，不切换旧 track/track-all，不迁 Web/bundle。

## 4. 接口与数据设计

应用用例先通过 repository 获取 due targets 和每账号 known content IDs，再逐账号调用
collector。成功结果保存实体并更新 `creator_fetched_at`；失败结果可保留实体但不更新时间；
评论成功时更新对应新帖 `comments_fetched_at`。watchlist 本身是配置资产，不建归属表。

## 5. 实现步骤

1. 新增 watchlist 请求、用例、Port 和独立输出。
2. 新增 XHS 两段式 collector：主页列表 → known diff → 新帖详情。
3. memory/MySQL 增 due、known、逐账号保存和成功标记。
4. 新增 YAML config、registry、container、CLI、日志与 spinner。
5. 新增增量、部分失败、search-only、用例互不 import、fixture E2E 和事务测试。

## 6. 测试策略

fixture 验证只抓未知帖；重复状态验证未到期跳过；MySQL 验证最旧优先、成功标记和失败
不标记；E2E 验证 auto_top_n 不产生账号；全量回归保护旧 track。

## 7. 验收标准

同目录 checklist 全绿；用户配置哈希不变；真实采集按用户要求不执行。

## 8. 风险与禁止事项

禁止读取 account/search 输出；禁止失败账号写 fetched_at；禁止本期替换旧 track；
禁止新增 watchlist 数据归属表或运行台账。

## 9. 回滚口径

删除新 watchlist 用例、平台 wrapper、输出和 CLI 即可；旧 track 入口未切换。

## 实施记录

- 已实现独立应用链、两段式增量、每账号 repository 调用、YAML/CLI、日志与输出。
- `refresh_days=0` 保持每轮参与；大于零时未到期跳过；batch 最旧优先。
- 主页列表和档案同时为空视为失败，不写成功标记。

## 实施偏差记录

- 终端当前以逐账号日志和 spinner 展示 watchlist；详情的双阶段条仍复用在旧 track，待平台
  gateway 完全迁移时再接入，避免平台层依赖 Rich。
- `run.sh track/track-all` 未切换，符合本期回滚边界。

## 蓝图验证备忘

三个一等用例互不 import；watchlist 不读取搜索榜单；共享的只有实体、平台原始采集能力
和 repository 基础设施。

## Gate 决策结果

- 用户确认各功能是类似后端独立接口，不能互相耦合。
- 用户要求暂不测试真实采集并继续实施。
