## 4. 接口与数据设计

**模块与依赖方向**（落地蓝图 §2，铁律：`pipelines → core → models`；`adapters` 实现 `core` 端口、由 `pipelines` 注入；`core` 不 import adapter、不含 "xhs"、不碰原始 JSONL）：

```
src/
  models.py                     # pydantic 模型（含 FetchResult）
  core/
    ports.py                    # ResearchAdapter(ABC)  —— 端口定义在 core
    keyword_expander.py
    aggregator.py
    account_ranker.py
    note_selector.py
    exporter.py
  adapters/
    parsers.py                  # xhs JSONL -> models（xhs 专属，归一化在此）
    fixture_adapter.py          # FixtureAdapter
  pipelines/
    run_research.py             # typer CLI: run_research()
configs/sample.yaml
```

**数据模型**（pydantic `BaseModel`，`.model_dump()` 出 dict）：

```python
class Account(BaseModel):
    account_id: str             # = user_id
    nickname: str
    source_keywords: list[str]  # 跨关键词合并去重（aggregator 填）
    note_count: int             # 派生：该账号相关 note 数（aggregator 填）
    first_seen_at: str          # v0 占位：该账号笔记 collected_at 最小值
    last_seen_at: str           # v0 占位：最大值

class Note(BaseModel):
    note_id: str
    account_id: str
    title: str
    body: str                   # <- desc
    tags: list[str]             # <- tag_list 逗号串拆分
    url: str                    # <- note_url
    like_count: int             # <- liked_count，归一化
    collect_count: int          # <- collected_count，归一化
    comment_count: int          # <- comment_count，归一化
    published_at: str           # <- time（毫秒 epoch，转 ISO 字符串）
    collected_at: str           # 运行时注入
    source_keywords: list[str]  # 单条来源；aggregator 阶段可并入更多
    raw_path: str               # fixture 文件路径（留痕）

class Comment(BaseModel):       # 期1 仅定义，不产数据
    body: str; note_id: str; like_count: int; collected_at: str

class AccountRank(BaseModel):
    account_id: str; nickname: str
    relevant_note_count: int; keyword_hit_count: int
    avg_interaction: float; account_score: float; note_ids: list[str]

class TypicalNote(BaseModel):
    account_id: str; note_id: str; title: str; url: str
    note_score: float; selection_reason: str

class FetchResult(BaseModel):
    provider: str; operation: str; collected_at: str
    keyword: str | None = None; page: int | None = None; note_id: str | None = None
    notes: list[Note] = []; accounts: list[Account] = []   # adapter 已解析；core 只读这两个
    raw_path: str | None = None; raw_text: str | None = None
    error: str | None = None; command: str | None = None
    @property
    def ok(self) -> bool: return self.error is None
```

**端口**（`core/ports.py`）：

```python
class ResearchAdapter(ABC):
    provider_name: str
    @abstractmethod
    def search(self, keyword: str, page: int, limit: int, collected_at: str) -> FetchResult: ...
    def fetch_comments(self, note_id: str, limit: int, collected_at: str) -> FetchResult:
        raise NotImplementedError   # 期3 才实现；期1/2 adapter 不必覆盖
```

**归一化规则**（`parsers.py`，在 adapter 边界）：

- 互动计数 `"1万" → 10000`、`"1.2万" → 12000`、`"10万+" → 100000`（取下界，去 `+`）、纯数字串 `"921" → 921`、空/异常 → `0`。
- `tag_list` 逗号串 → `list[str]`（去空白；空串 → `[]`）。
- `time`（毫秒 epoch）→ ISO `str`（UTC）。

**核心算法**（v0 占位，蓝图允许各期演进，**不算偏差**；权重落 config）：

- `keyword_expander`：读 config `keywords` + 可选 `synonyms` 映射，输出去重后的关键词列表。**synonyms 为 v0 占位扩展策略，允许各期演进**。
- `aggregator`：把多个 `FetchResult.notes` 聚成 `(notes, accounts)`——按 `account_id` 去重生成 `Account`，`note_count`=该账号笔记数，`source_keywords`=其笔记来源关键词并集；同 `note_id` 跨关键词重复时合并 `source_keywords`、保留一条。
- `account_ranker`：`interaction(note)=like+collect+comment`；`avg_interaction`=账号笔记均值；`account_score = w_note*note_count + w_hit*keyword_hit_count + w_inter*avg_interaction`（权重 config，默认 `10/5/0.01`）。
- `note_selector`：`note_score = like + 2*collect + 3*comment`；每账号取 `top_notes_per_account`（默认 2）条，`selection_reason` 记 "top by interaction"。
- `exporter`：输出下列稳定列；list 字段以 `|` 连接。

**导出契约**（文件名与列稳定）：

| 文件 | 列 |
|---|---|
| accounts.csv | account_id, nickname, source_keywords, note_count, first_seen_at, last_seen_at |
| notes.csv | note_id, account_id, title, body, tags, url, like_count, collect_count, comment_count, published_at, collected_at, source_keywords, raw_path |
| account_rank.csv | account_id, nickname, relevant_note_count, keyword_hit_count, avg_interaction, account_score, note_ids |
| typical_notes.csv | account_id, note_id, title, url, note_score, selection_reason |
| report_input.md | 按 account_score 降序的账号清单 + 每账号典型笔记（供 LLM） |

**config 形状**（`configs/sample.yaml`）：

```yaml
provider: fixture
fixture_path: tests/fixtures/search_contents_sample.jsonl
keywords: ["留学辅导"]
synonyms: { 留学辅导: ["essay辅导", "final自救"] }
search: { pages: 1, limit: 20 }
ranking: { weights: { note_count: 10, keyword_hit: 5, interaction: 0.01 } }
selection: { top_notes_per_account: 2 }
export: { out_dir: data/exports }
```
