### 质疑1【已关闭】：`export_all(comments=[])` 是否会产 `comments.csv`？"有评论时才写"与方案"开评论产6文件"是否矛盾？

**可证伪点**：若 `export_all(comments=[])` 产出 `comments.csv`（即使空），则实施行为=方案文字字面（开评论→6文件）；若不产，则存在"开评论但0评论→5文件"路径，与方案§5步骤4 单测注释"e2e 开评论→6文件"有语义距离。

**验证**：亲跑 `uv run python -c "…export_all(…, comments=[], …)"` → `paths.keys()` 不含 `"comments"`，`comments.csv` 不存在。实施用 `if comments:` 判断（非空列表才写）。

**方案文本核查**：
- 方案§2 After："多产 `comments.csv`"——未说"有评论才多产"
- 方案§5 步骤4 单测："e2e fixture 开评论→6文件"——fixture 有 2 条评论，测试确实跑出6文件
- 方案§5 目标5："未启用/不支持/采集失败→评论空、管线续、其余5文件不变"——fetch_comments 成功返回空列表≠失败，未被此规则覆盖

**结论**：方案存在一处口径空白——`enabled:true` 且 `fetch_comments` 成功但返回 0 条评论时，方案未明确产 5 还是 6 文件。实施选择"有评论才写 CSV"（不产空文件），合理但未与方案明确对齐。在正常使用场景（fixture/真实评论均有 ≥1 条）不影响行为；§7 验收"产6文件"通过。**此质疑不构成阻塞**（方案未明文禁止此实现），记为非阻塞建议，由设计裁量。

### 质疑2【已关闭】：`export_all` 的 `comments: list[Comment] = []` 是 mutable default arg，是否有潜在 bug？

**可证伪点**：若 `export_all` 在某调用路径中修改了 `comments` 参数，则共享 default 对象会导致跨调用污染，表现为后续调用看到上次遗留的评论。

**验证**：亲读 `src/core/exporter.py:62-175`，`export_all` 仅遍历 `comments`（读取），无 `append`/`extend`/`clear` 等修改操作。ruff B006 规则虽命中，但当前 ruff 配置 `select = ["E","F","I","W"]` 不含 B，不阻塞 lint。

**结论**：`export_all` 只读参数，mutable default 无实际 bug 影响。属 code smell 而非功能缺陷。**此质疑不构成阻塞**，记为非阻塞建议（改为 `comments: list[Comment] | None = None`，内部 `if comments is None: comments = []`）。

### 质疑3【已关闭】：`FixtureAdapter.fetch_comments` 中 `if limit: comments = comments[:limit]` 在 `limit=0` 时不截断，是否影响行为？

**可证伪点**：若 pipeline 传 `limit=0`，则 `FixtureAdapter` 返回 fixture 全部评论（不截断），但 `MediaCrawlerAdapter` 会传 `--max_comments_count_singlenotes 0` 给 MediaCrawler（行为取决于 MediaCrawler 实现），两者语义可能不一致。

**验证**：亲跑 `FixtureAdapter(…, comments_path=COMMENTS).fetch_comments([], 0, "2026")` → 返回 2 条（fixture 全部）。检查 `configs/sample.yaml`：`limit: 10`；`run_research.py`：`cc.get("limit", 10)`——默认 10，正常使用路径 limit≥1，`if limit:` 为 True 正常截断。

**结论**：limit=0 是使用层面不应出现的值，方案未规定其语义。正常配置不触发此路径。**此质疑不构成阻塞**，记为非阻塞观察。

### 质疑4【已关闭】：实施偏差"无"是否漏报了"有评论时才写 comments.csv"这一实施行为与方案字面的差异？

**可证伪点**：若方案明确写"开评论=6文件"而实施写"有评论才产第6文件"，则"无偏差"失实。

**验证**：查方案§5 步骤4 原文"单测：e2e fixture 开评论→6文件"以及方案§7"demo（fixture，开评论）退出0并产6文件"——这两条均在有 fixture 评论（2条）的语境下写成；方案§5 目标5 "采集失败→5文件"之外，"成功但0条评论"路径方案未覆盖。实施记录第4条写"有评论时新增 comments.csv"——此措辞本身也明确了行为，与"无偏差"不矛盾（方案口径模糊处由实施合理填充）。

**结论**：不构成失实偏差记录。偏差记录"无"属实（方案对此边界未明确，非实施偏离方案明文要求）。**此质疑已关闭**。
