# 小红书竞品研究工具 期3 方案 v1

## 给用户的摘要

期3（可选）补齐**典型笔记评论**：管线选出典型笔记后，对**这批笔记**（不是全部笔记）用 MediaCrawler 的 `detail` 模式**一次性批量**读评论 → 解析 → 导出。评论**只存 `body / note_id / like_count / collected_at` 四个字段**，评论用户身份（user_id/nickname/头像/IP）在 parser 边界即丢弃，结构上落死红线。

- Before：导出 5 文件（accounts/notes/account_rank/typical_notes.csv + report_input.md），无评论。
- After：多产 `comments.csv`，`report_input.md` 每条典型笔记下并入前几条高赞评论；`comments.enabled: false` 或 adapter 不支持时行为与期2 完全一致（零评论、5 文件不变）。

合规：仅对**典型笔记**（默认每账号 2 条）批量读一次评论，不对全量笔记抓评论——低频、增量最小。CDP 本机浏览器沿用期2。

拍板清单：

- **评论范围＝典型笔记（推荐）** vs 全量笔记：推荐仅典型笔记，请求量最小、契合低频自律；全量会显著增加采集动作。**默认按推荐执行，除非你要改。**
- 其余无新增（敏感字段口径、CDP、代理池边界已在蓝图 v5 + 期2 拍定）。

## 3. 目标 / 非目标

**目标**：

1. `parsers`：`parse_comments_jsonl_lines(lines, collected_at) -> list[Comment]`，只映射四字段，边界丢身份字段。
2. `MediaCrawlerAdapter.fetch_comments(typical_notes, limit, collected_at) -> FetchResult`：detail 模式批量传典型笔记 URL、一次 subprocess、读回 `*_comments_*.jsonl`、复用 parser；失败装 `error` 不抛。
3. `FixtureAdapter.fetch_comments(...)`：读本地评论 fixture JSONL，供离线管线与 CI 覆盖导出/织入路径。
4. `exporter`：新增 `comments.csv`（四列）+ `report_input.md` 每典型笔记下并入前 K 条高赞评论。
5. `pipeline`：选典型后按 `comments.enabled` 调 `fetch_comments`，产出并入导出；未启用/不支持/采集失败 → 评论空、管线续、其余 5 文件不变。
6. 单测（全 offline）+ 集成验收脚本（manual，真实跑一次典型笔记评论）。

**非目标**：不抓全量笔记评论；不抓二级/子评论；不存评论用户任何身份/IP/头像/图片字段；不改 `accounts/notes/account_rank/typical_notes.csv` 列与 `search`/`aggregate`/`rank`/`select` 逻辑；真实采集不进 CI；不做情感分析/聚类（留给下游 LLM）。

**`adapters/mediacrawler_adapter.py`（新增 `fetch_comments`）**：

```python
def fetch_comments(self, notes, limit, collected_at) -> FetchResult:
    save_path = self._save_path(collected_at + "-comments")   # 与 search 产出隔离
    urls = [n.url for n in notes if n.url]
    cmd = [*launcher, "main.py", "--platform","xhs", "--type","detail",
           "--specified_id", ",".join(urls),
           "--get_comment","yes", "--get_sub_comment","no",
           "--max_comments_count_singlenotes", str(limit),
           "--save_data_option","jsonl", "--save_data_path", str(save_path),
           "--enable_ip_proxy","no", "--max_concurrency_num","1", "--lt", login_type]
    # 复用 _run_crawler / _err；读回 glob xhs/jsonl/*_comments_*.jsonl → parse_comments_jsonl_lines
    # 无 urls → 直接返回空 comments 的 ok 结果（不起浏览器）
```

## 6. 测试策略

**单测（offline，进 CI）**：

- **红线（行为断言，最重要）**：`parse_comment` 输入含 `user_id/nickname/avatar/ip_location` 的真实评论行，输出 `Comment` 只有四字段；`comments.csv` 表头 `== ["body","note_id","like_count","collected_at"]`；grep 断言导出文本不含被采样身份值。
- 命令构造：`fetch_comments` 命令含 `--type detail`、`--specified_id <url1,url2>`、`--get_comment yes`、`--get_sub_comment no`、`--enable_ip_proxy no`；空 urls 不构命令/不起子进程。
- 读回解析：真实评论样本摆 `{save_path}/xhs/jsonl/detail_comments_*.jsonl` → `fetch_comments` 出 ≥1 Comment。
- 失败/降级：subprocess 非 0 → `FetchResult.error`、`.comments==[]`、管线续；`NotImplementedError`（fixture 无评论 fixture 时）被 pipeline 捕获。
- 回归：`comments.enabled: false` → 导出 5 文件、列与期2 逐字节同构（不破期1/2）。

**集成验收（manual，不进 CI）**：登录后跑扩展脚本，真实对典型笔记取评论 → 看 `comments.csv` 有真实评论、`report_input.md` 织入、**人工核对无用户昵称/ID 列**。盲区：真实评论量/风控稳定性、xsec_token 时效——标注。

## 8. 风险与禁止事项 / 回滚口径

- **身份字段红线**：评论只存四字段；结构上由 parser 只映射三个源字段 + Comment 模型仅四字段兜底；测试与 checklist 双重守。**任何时候不得为"分析方便"加回 user_id/nickname。**
- **xsec_token 时效**：detail 模式依赖 `Note.url` 的 xsec_token，采集与 search 间隔过久可能失效 → 表现为评论空/error，管线续、不假装；建议同一次运行内 search→评论连贯执行。
- **采集失败不假装**：`fetch_comments` 失败记 error、评论空，5 主文件照出（诚实口径，沿用期2）。
- **真实采集不可测**：CI 只跑 fixture；真实评论靠 manual 集成验收。
- **禁止项（红线）**：不抓全量/二级评论、不存评论用户身份/IP/头像/图片、不开付费代理池、禁并发(>1)、`core` 禁 "xhs"。
- **回滚**：未合主线前期3 全在 `feat/phase3-comments`，弃分支即回期2；`FetchResult.comments`/`export_all(comments=)` 为默认空 additive，回滚不影响期1/2。
