# 小红书竞品研究工具 期2 方案 v1

## 给用户的摘要

期2 接入真实数据源 **MediaCrawler**：新增一个 `mediacrawler_adapter`，触发 MediaCrawler 跑小红书关键词搜索 → 读回它产的 JSONL → **复用期1 `parsers`** → `FetchResult`；MediaCrawler 不可用/采集失败时降级 fixture，管线不中断。`core`/`models`/`parsers`/`exporter` **一行不改**。

- Before：管线只能吃离线 fixture。
- After：`provider: mediacrawler` 时真实采集（关 CDP/代理池、单并发、低频）；`provider: fixture` 或 MediaCrawler 不可用时走离线样本。

拍板清单：**无新增**（数据源、禁用边界已在蓝图 v5 + 需求回写拍定）。

## 1. 起点

`起点 = main HEAD`（期1 已合入，merge commit `c3bc97e`）。新建功能分支 `feat/phase2-mediacrawler`。一切 diff 以此为参照。

## 2. 现状与约束盘点

**复用盘点**（期1 成果，期2 直接复用、不改）：

- `src/models.py`、`src/core/*`（ports/aggregator/ranker/selector/exporter/keyword_expander）、`src/adapters/parsers.py`、`src/adapters/fixture_adapter.py`、`src/pipelines/run_research.py`——全部复用。
- **期2 仅新增 `src/adapters/mediacrawler_adapter.py` + config 扩展 + pipeline 的 provider 选择分支 + 集成验收脚本**。

**MediaCrawler 调用实测**（spike，带 `文件:行号`，位置 `../MediaCrawler/`）：

- 命令行参数：`cmd_arg/arg.py:160-334`——`--platform/--type/--keywords/--lt/--cookies/--save_data_option/--save_data_path/--enable_ip_proxy/--crawler_max_notes_count/--max_concurrency_num`。
- 关键 config：`config/base_config.py` — `PLATFORM:21`、`KEYWORDS:27`、`LOGIN_TYPE:28`、`ENABLE_IP_PROXY:34`(默认 False)、`ENABLE_CDP_MODE:59`(默认 **True**，注释自承降风控)、`CDP_CONNECT_EXISTING:83`、`SAVE_DATA_OPTION:90`(默认 jsonl)。
- 入口：`main.py:100-110` `async main()`：`cmd_arg.parse_cmd()` 覆盖 config → `CrawlerFactory.create_crawler` → `crawler.start()`。
- **CLI 无 CDP 开关**：禁用 CDP 只能 import 设 `config.ENABLE_CDP_MODE=False`（参考 `../data/.mediacrawler_xhs_runner.py` 的 import+config+asyncio 模式）。

**约束**：

- 合规（需求/蓝图 v5）：调用时**显式** `ENABLE_IP_PROXY=False`、`ENABLE_CDP_MODE=False`、`MAX_CONCURRENCY_NUM=1`、低频；不存评论用户敏感字段（期2 不碰评论）。
- 真实采集要 MediaCrawler 自身环境（Playwright 浏览器）+ 登录态；**不可进 CI**。
- 只读边界：仅 search 抓取，无互动写。

## 3. 目标 / 非目标

**目标**（期2 交付）：

1. `MediaCrawlerAdapter`（实现 `ResearchAdapter`）：以 subprocess 跑一个 runner（import 设合规 config + `main()`）触发 MediaCrawler search，读回 JSONL，复用 `parsers` 出 `FetchResult`；失败装 `error` 不抛。
2. pipeline `provider` 选择：`fixture` | `mediacrawler`；MediaCrawler 目录不存在 → 启动降级 fixture（日志提示）。
3. config 扩展（`provider/mediacrawler_dir/login_type/cookies/out_dir/max_notes` 等）+ `configs/sample_mediacrawler.yaml`。
4. 单测（全 offline）+ 集成验收脚本（manual，真实跑一次）。

**非目标**：不改 `core/models/parsers/exporter` 契约；不做评论（期3）；不并发/代理池/CDP 反风控；真实采集不进 CI；不做 MediaCrawler 环境的自动安装。

## 4. 接口与数据设计

**新增 `src/adapters/mediacrawler_adapter.py`**（依赖方向不变：adapter 实现 core 端口、由 pipeline 注入；core 仍 0 处 "xhs"）：

```python
class MediaCrawlerAdapter(ResearchAdapter):
    provider_name = "mediacrawler"
    def __init__(self, mediacrawler_dir: str, out_dir: str, *,
                 login_type: str = "qrcode", cookies: str = "",
                 max_notes: int = 20, python_exe: str = "uv"): ...
    def search(self, keyword, page, limit, collected_at) -> FetchResult:
        # 1. 组装 runner：import config; 设 PLATFORM=xhs, CRAWLER_TYPE=search,
        #    KEYWORDS=keyword, ENABLE_IP_PROXY=False, ENABLE_CDP_MODE=False,
        #    MAX_CONCURRENCY_NUM=1, SAVE_DATA_OPTION=jsonl, SAVE_DATA_PATH=out_dir,
        #    LOGIN_TYPE=login_type, COOKIES=cookies, START_PAGE=page,
        #    CRAWLER_MAX_NOTES_COUNT=limit; import main; asyncio.run(main.main())
        # 2. subprocess 跑 runner，cwd=mediacrawler_dir，用其环境（uv run --project 或其 .venv）
        # 3. 成功→读 out_dir 最新 search_contents_*.jsonl，复用 parsers.parse_jsonl_lines
        #    →FetchResult(notes, accounts, raw_path, command)
        # 4. 失败(非0/超时/无输出/异常)→FetchResult(error=..., command=...)
```

合规开关在 runner 里**硬编码为禁用值**（不可由 config 放开），作为机械可核验的禁用点（预飞 grep）。

**pipeline 改动**（`run_research.py` `_build_adapter`）：

```python
provider = config.get("provider", "fixture")
if provider == "mediacrawler":
    mc_dir = config["mediacrawler_dir"]
    if Path(mc_dir).exists():
        return MediaCrawlerAdapter(mc_dir, out_dir=..., login_type=..., cookies=...)
    log("MediaCrawler 目录不存在，降级 fixture")   # 降级不中断
return FixtureAdapter(config["fixture_path"])
```

**契约不变**：`Note/Account/FetchResult`、`parsers`、导出 5 文件全沿用期1。`raw` 落 `data/raw/`。

## 5. 实现步骤

1. `mediacrawler_adapter.py`：runner 命令/脚本组装（含合规开关硬编码）+ `search` 骨架（失败装 error）；单测验证**命令构造含 `ENABLE_IP_PROXY=False`/`ENABLE_CDP_MODE=False`/`MAX_CONCURRENCY_NUM=1`**、不实跑。
2. 读回解析：跑完读 out_dir 最新 jsonl → 复用 `parsers`；单测把现有真实样本 `tests/fixtures/` 放进临时 out_dir，验证出 notes/accounts。
3. 失败/降级路径：subprocess 非 0/超时/无输出 → error FetchResult；单测覆盖。
4. pipeline `provider` 选择 + config 扩展 + `configs/sample_mediacrawler.yaml`；`provider: fixture` demo 仍跑通（不破坏期1）。
5. 集成验收脚本 `scripts/integration_mediacrawler.py`（manual：真实跑一次 search→管线→5 导出，需登录）+ README 用法。

## 6. 测试策略

**单测（全 offline，进 CI）**：

- 命令构造：`MediaCrawlerAdapter` 组装的 runner/命令**含合规禁用开关**、关键词/输出路径正确（行为断言）。
- 读回解析：把真实样本当 MediaCrawler 输出放 out_dir → `search` 复用 parsers 出 ≥1 note（验证复用闭环）。
- 失败降级：mediacrawler_dir 不存在 → pipeline 降级 fixture；subprocess 失败 → `FetchResult.error` 且 `.ok=False`。
- 回归：`provider: fixture` 端到端仍产 5 文件（期1 不破）。

**集成验收（manual，不进 CI）**：登录态在时跑 `scripts/integration_mediacrawler.py`，真实 search→JSONL→管线→5 导出，人工看 `report_input.md` 合理。盲区：真实采集稳定性（风控）、MediaCrawler 环境差异——标注。

## 7. 验收标准 + 检查清单 YAML

人话（机械参数落 `checklist.yaml`）：

- **必过命令**：`uv run ruff check .`/`ruff format --check .`/`pytest -q` 全绿；demo `provider: fixture` 退出 0（保证不破期1）。
- **必存产物**：`configs/sample_mediacrawler.yaml`、`scripts/integration_mediacrawler.py`。
- **必含符号**：`class MediaCrawlerAdapter`(src/adapters)。
- **禁留标记**：`src/core` 无 `xhs`；全仓 `src` 无 `print(`；**adapter 内合规开关无被设 True**（grep `ENABLE_IP_PROXY = True`/`ENABLE_CDP_MODE = True` 0 命中）。
- 集成验收（真实采集）**单列 manual，不入必过命令**（不可自动化，见 §6）。

## 8. 风险与禁止事项

- **关 CDP 守合规 ↔ 采集成功率**：禁用 CDP 反风控用标准 Playwright，真实采集**更易被风控拦/失败**。本期接受此代价（守需求"不绕风控"）；若实测频繁失败，作为变更协议另议，不在期2 内放宽。
- **MediaCrawler 环境依赖**：需其 Playwright 环境与登录态；adapter 以 subprocess 用其环境跑，环境缺失→降级 fixture。
- **真实采集不可测**：CI 只跑 offline 单测；真实跑通靠 manual 集成验收。
- **禁止项（红线）**：禁开 `ENABLE_IP_PROXY`/`ENABLE_CDP_MODE`、禁并发(>1)、禁评论互动写、`core` 禁 "xhs"。
