## 可证伪质疑

1. **【阻塞】「采集失败→降级 fixture」名实不符。** 证据:`aggregator.py:19` 对 `not r.ok` 直接 `continue`,§4 失败路径返回 error FetchResult 而非重建 FixtureAdapter。故「目录在、跑挂」时实际产**空输出**,不回退 fixture。请二选一:(a)改摘要/§3 文字为「目录不存在才降级 fixture;采集失败装 error、该次产空、管线不中断」;或(b)若确需失败也回退 fixture,在 adapter 或 pipeline 显式实现并加单测。当前文档自相矛盾,未关闭=阻塞。
2. **【阻塞】最新 jsonl 定位钉得不够,按字面实现会读不到文件。** 证据:`async_file_writer.py:39,43`——真实路径 `<out_dir>/xhs/jsonl/search_contents_<日期>.jsonl`,非 `<out_dir>/*.jsonl`;且日期命名+`'a'` 追加(line 59),同日多跑同文件累积。§4「读 out_dir 最新 search_contents_*.jsonl」未含 `xhs/jsonl/` 子路径、未交代同日追加如何取本跑数据。请在 §4 钉死 glob 路径与「同日追加」的截断/去重口径(并据此修步骤 2 单测的样本投放目录)。未关闭=阻塞。
3. **【非阻塞】`python_exe="uv"` 与伪码 `python main.py` 不一致。** 证据:MediaCrawler README 用 `uv run main.py`,其有独立 `.venv`。默认走 `uv run` 正确,但 §4 注释写 `python main.py` 易误导实施。建议统一为 `uv run main.py …` 并注明 cwd=mediacrawler_dir、unset VIRTUAL_ENV。可在 v2 顺手澄清,不单独阻塞。
4. **【非阻塞·待实施验证】page→`--start`、limit→`--crawler_max_notes_count` 的语义映射** 在 pipeline `for page in 1..N` 循环下是否产生预期分页,只能 manual 验;offline 不可证。已被 §6 盲区覆盖,登记备查。
