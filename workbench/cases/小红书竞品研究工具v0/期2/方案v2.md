# 小红书竞品研究工具 期2 方案 v2

## 给用户的摘要

期2 接入真实数据源 **MediaCrawler**：新增 `mediacrawler_adapter`，命令行触发 MediaCrawler 跑小红书关键词搜索 → 读回它产的 JSONL → **复用期1 `parsers`** → `FetchResult`。`core`/`models`/`parsers`/`exporter` **一行不改**。

- Before：管线只能吃离线 fixture。
- After：`provider: mediacrawler` 时真实采集（CDP 用本机浏览器、关代理池、单并发、低频）。**MediaCrawler 目录/环境不可用 → 启动降级 fixture（管线仍跑通）；采集运行失败 → 记 error、管线不中断（产出可能空），不用过时数据冒充真实。**

拍板清单：**无新增**（数据源、合规口径已在蓝图 v5 + 需求拍定，CDP 个人自用允许）。

## v1→v2 变更（供差量评审）

回应「期2 方案评审 #1」（verdict 需修改，2 阻塞）：

1. **降级语义对齐名实**（阻塞①）：明确**两条不同路径**——(a)「MediaCrawler 目录/环境不可用」在 pipeline `_build_adapter` 启动时切回 `FixtureAdapter`（真降级）；(b)「目录在但采集运行失败」由 `search` 返回 `FetchResult(error=...)`，`aggregator` 既有语义跳过该结果、管线续（产出可能空），**不回退 fixture、不假装**。摘要/§3/§4 文字据此校正，删去「失败降级 fixture」的夸大表述。
2. **钉死输出路径 + 同日追加隔离**（阻塞②）：MediaCrawler 实际写 `{SAVE_DATA_PATH}/xhs/jsonl/search_contents_<日期>.jsonl` 且**追加写**（`../MediaCrawler/tools/async_file_writer.py:37-44,59`）。adapter 每次 `search` 用**唯一 save_path**（`<out_dir>/<collected_at 安全化>/`）隔离本次产出，读 `glob('{save_path}/xhs/jsonl/search_contents_*.jsonl')`，避免历史行混入。
3. 其余（命令行直调、复用 parsers、合规 CDP 允许/关代理池）沿用 v1。

## 3. 目标 / 非目标

**目标**：

1. `MediaCrawlerAdapter`（实现 `ResearchAdapter`）：subprocess 命令行触发 MediaCrawler search，读回唯一 save_path 下 JSONL，复用 `parsers` 出 `FetchResult`；失败装 `error` 不抛。
2. pipeline `provider` 选择：`fixture` | `mediacrawler`；**mc 目录/环境不可用 → 启动降级 `FixtureAdapter`**；采集运行失败 → `FetchResult.error`，aggregator 跳过、管线续（不回退 fixture）。
3. config 扩展（`provider/mediacrawler_dir/login_type/cookies/out_dir/max_notes`）+ `configs/sample_mediacrawler.yaml`。
4. 单测（全 offline）+ 集成验收脚本（manual，真实跑一次）。

**非目标**：不改 `core/models/parsers/exporter` 契约；不做评论（期3）；不开付费代理池、不并发；真实采集不进 CI；不自动装 MediaCrawler 环境。

## 4. 接口与数据设计

**新增 `src/adapters/mediacrawler_adapter.py`**（依赖方向不变；core 仍 0 处 "xhs"）：

```python
class MediaCrawlerAdapter(ResearchAdapter):
    provider_name = "mediacrawler"
    def __init__(self, mediacrawler_dir: str, out_dir: str, *,
                 login_type: str = "qrcode", cookies: str = "",
                 max_notes: int = 20): ...
    def search(self, keyword, page, limit, collected_at) -> FetchResult:
        # 1. 本次唯一输出目录隔离同日追加：
        #    save_path = <out_dir>/<collected_at 安全化为目录名>
        # 2. subprocess 跑 MediaCrawler 命令行（cwd=mediacrawler_dir，用其环境）：
        #    python main.py --platform xhs --type search --keywords <keyword>
        #      --lt <login_type> [--cookies ...] --save_data_option jsonl
        #      --save_data_path <save_path> --enable_ip_proxy no
        #      --max_concurrency_num 1 --crawler_max_notes_count <limit> --start <page>
        # 3. 成功→ glob('{save_path}/xhs/jsonl/search_contents_*.jsonl') 读本次产出，
        #    复用 parsers.parse_jsonl_lines →FetchResult(notes, accounts, raw_path=save_path, command)
        # 4. 失败(非0/超时/无输出/异常)→FetchResult(error=..., command=...)
```

合规（个人自用）：代理池 `--enable_ip_proxy no`、并发 `--max_concurrency_num 1`；CDP 用 MediaCrawler 默认（本机真实 Chrome，不伪造身份故允许）。全部经命令行设定，无需 runner 改 config。

**pipeline 改动**（`run_research.py` `_build_adapter`）：

```python
provider = config.get("provider", "fixture")
if provider == "mediacrawler":
    mc_dir = config["mediacrawler_dir"]
    if Path(mc_dir).exists():
        return MediaCrawlerAdapter(mc_dir, out_dir=config[...], login_type=..., cookies=...)
    log("MediaCrawler 目录不存在，降级 fixture")     # 真降级：路径 (a)
return FixtureAdapter(config["fixture_path"])
# 采集运行失败＝路径 (b)：MediaCrawlerAdapter.search 返回 error FetchResult，
#   aggregator 既有逻辑跳过 not ok 结果、管线续（产出可能空），不回退 fixture。
```

**契约不变**：`Note/Account/FetchResult`、`parsers`、导出 5 文件全沿用期1。

## 5. 实现步骤

## 6. 测试策略

**单测（offline，进 CI）**：

- 命令构造：`MediaCrawlerAdapter` 组装的命令含 `--enable_ip_proxy no --max_concurrency_num 1`、关键词、唯一 save_path（行为断言）。
- 读回解析：把真实样本摆成 `{save_path}/xhs/jsonl/search_contents_*.jsonl` → `search` 复用 parsers 出 ≥1 note。
- 失败/降级：mc_dir 不存在 → pipeline 降级 fixture（路径 a）；subprocess 失败 → `FetchResult.error`、`.ok=False`、aggregator 跳过（路径 b）。

**集成验收（manual，不进 CI）**：登录态在时跑 `scripts/integration_mediacrawler.py`，真实 search→JSONL→管线→5 导出，人工看 `report_input.md`。盲区：真实采集稳定性、MediaCrawler 环境差异——标注。

## 8. 风险与禁止事项

- **MediaCrawler 环境依赖**：需其 Playwright 环境与登录态；环境缺失→启动降级 fixture（路径 a）。
- **采集失败不假装**：采集运行失败记 error、管线产出可能空，**不用过时 fixture 冒充真实**（路径 b）；这是诚实口径，演示场景请显式用 `provider: fixture`。
- **真实采集不可测**：CI 只跑 offline 单测；真实跑通靠 manual 集成验收。
- **采集成功率**：CDP（本机真实浏览器）提高成功率；个人自用、低频。
- **禁止项（红线）**：不开付费 IP 代理池（`--enable_ip_proxy no`）、禁并发(>1)、禁评论互动写、`core` 禁 "xhs"。
