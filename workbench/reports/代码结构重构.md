## 背景

三期功能陆续焊接后，几个大文件出现样板堆积与裸 dict 配置的「屎山感」。全仓扫描（19 个源文件、约 2980 行）判断：架构本身健康（ports/adapters/core/pipelines 六边形分层守得住、core 平台无关、models 干净），问题集中在少数文件的**局部重复**与**魔法字符串配置**，宜外科手术式清理，不推倒重来。

重灾区：`exporter.py`(437)、`mediacrawler_adapter.py`(678)、以及散落在 `run_research.py`/`cli.py` 的配置访问与跨模块私有耦合。

## 目标与非目标

- 目标：消除样板重复、给配置上类型、解开 cli↔run_research 的私有耦合，**全程零行为变更**、导出契约逐字节不变。
- 非目标：不动 `core/` 计算逻辑、不动 ports 抽象与 models、不改导出契约的列名/顺序/数值格式、不做过度工程（progress 两个进度工厂虽同构但未强行合并）。

## 五项改动

1. **exporter CSV schema 化**：9 张表各自「header 列表 + 逐字段 row 推导式」的三重字段复写 → 每表一处 `*_COLUMNS`（列名→取值函数）声明 + 通用 `_write_model_csv`；`notes`/`creator_notes` 共用 `NOTE_COLUMNS`（原为逐字复制的 13 列）。437→367 行。
2. **adapter 公共 flag + 读回收敛**：三命令构建器抽出共享 `_base_command`/`_append_cookies`；四个 `_read_*` 抽出 `_read_jsonl(save_path, pattern)`。678→649 行，重复消除为主。
3. **config → pydantic RunConfig**：新增 `src/pipelines/config.py`（RunConfig + 9 个子段模型）；入口一次解析（`_resolve_config_refs` 裸 dict 预处理 → `model_validate`），下游 `_build_adapter`/三个 stage 全程属性访问，`.get("x",{}).get("y",D)` 深挖链清零。保留原容错口径（缺段走默认、未知键 extra=ignore）。
4. **解耦 cli↔run_research**：新增 `src/pipelines/runtime.py` 收编排工具（`now_iso`/`build_adapter`/`resolve_config_refs`/`update_latest_link`/`compact_run_id`/`prepare`）；cli 与 run_research 都以 `runtime.X` 模块属性访问，cli 不再读 run_research 私有符号。
5. **进度回调样板**：新增 `_attach_progress(adapter, on_progress)` 上下文管理器，替换 `_search_stage`/`_sync_stage` 里重复的「注入回调→finally 复位」块。

## 顺带修掉的两个真问题

- **数据陷阱**（config 迁移引入并当场修复）：`provider=mediacrawler` 但缺 `mediacrawler_dir` 时，初版会静默降级 fixture（真采集悄悄变假数据）→ 改为显式 `ValueError`。真实「本机无 MediaCrawler」降级走的是「键有值但路径不存在」那条，不受影响。
- 两处文档断链：`config.py` 注释里 `_resolve_config_refs` 旧路径 → `runtime.resolve_config_refs`。

## 口径变化（后续注意）

monkeypatch pin 锚点从 `src.pipelines.run_research._now_iso` 迁到 **`src.pipelines.runtime.now_iso`**（`_build_adapter` → `runtime.build_adapter`）。测试里 18+ 处 pin 已随迁；以后 pin 时间源/adapter 钉 `runtime.*`。

## 验收

- 五项各派子代理代码评审，均确认**零行为偏差**（逐列核对导出契约、逐 flag 核对命令、默认值逐字段核对、pin 传播核对）。
