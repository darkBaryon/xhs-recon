# 小红书竞品研究工具 v0

## 背景

个人自用、本地运行的小红书竞品研究工具。给定 seed keywords（如「留学生课程辅导」「essay辅导」「final自救」），跑一条本地研究管线，产出供 LLM 分析的结构化资料。**这套能力预期复用到后续功能**，因此按工程化方式做：采集层与研究核心解耦，接口稳定可调用，不写一次性脚本。

## 目标

给定 seed keywords，本地管线完成：

1. 扩展关键词；
2. 调用本地 `xhs` CLI 做只读搜索，保存 raw JSON；
3. 从搜索结果提取 notes 和 accounts；
4. 对账号去重、打分；
5. 为每个高相关账号选典型笔记；
6. （可选）读取典型笔记评论；
7. 导出结构化文件（CSV / JSONL / Markdown）供 LLM 分析。

## 非目标

- 个人自用、本地运行；采集调用现成 MediaCrawler，不自研爬虫；
- 不做自动点赞 / 评论 / 私信等互动写操作；
- 不为历史版本 / 旧前端 / 旧数据库做兼容。

## 定档提议

**蓝图级**。理由：虽个人自用，但明确要**跨期复用**——采集层（xhs 专属）与研究核心（平台无关）需要稳定接口契约；且本机暂无 `xhs` 命令，「fixture 管线」与「真实 CLI 接入」天然分期、各期都够标准级。需要先出薄蓝图钉模块边界与稳定契约，再逐期走标准级。

## 验收口径

- `python -m src.pipelines.run_research --config configs/sample.yaml` 端到端跑通；
- 无 `xhs` 命令时自动降级 fixture，全流程不中断；
- 产出约定的导出文件（accounts / notes / account_rank / typical_notes / report_input.md）；
- 测试不依赖真实小红书访问（吃 `tests/fixtures/` 样本）；
- 导出与 fixture 中**不含评论用户敏感字段**。
